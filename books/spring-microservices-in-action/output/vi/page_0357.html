332 PHỤ LỤC A Chạy cloud trên desktop của bạn 3

Nó đẩy image Docker lên kho lưu trữ image Docker cục bộ được cài khi bạn cài Docker.

Listing sau đây trình bày nội dung Dockerfile từ licensing service của bạn.

Trong Dockerfile từ listing này, bạn đang cấp phát instance bằng Alpine Linux (https://alpinelinux.org/). Alpine Linux là một bản phân phối Linux nhỏ thường dùng để build image Docker. Image Alpine Linux mà bạn dùng đã có Java JDK cài sẵn.

Khi cấp phát image Docker, bạn sẽ cài một tiện ích dòng lệnh tên là nc. Lệnh nc dùng để ping một máy chủ và kiểm tra một cổng cụ thể có đang trực tuyến không. Bạn sẽ dùng nó trong script lệnh run.sh để đảm bảo trước khi khởi chạy dịch vụ, tất cả dịch vụ phụ thuộc của nó (ví dụ, cơ sở dữ liệu và dịch vụ Spring Cloud Config) đã khởi động. Lệnh nc làm điều này bằng cách theo dõi các cổng mà dịch vụ phụ thuộc lắng nghe. Việc cài nc được thực hiện qua lệnh RUN apk update && apk upgrade && apk add netcat-openbsd, chạy các dịch vụ bằng Docker Compose.

Tiếp theo, Dockerfile sẽ tạo một thư mục cho file jar thực thi của licensing service rồi sao chép file jar từ hệ thống tệp cục bộ sang thư mục đã tạo trên image Docker. Tất cả được thực hiện qua lệnh ADD licensing-service-0.0.1-SNAPSHOT.jar /usr/local/licensingservice/.

Bước tiếp theo trong quy trình cấp phát là cài script run.sh qua lệnh ADD. Script run.sh là script tùy chỉnh tôi viết để khởi chạy dịch vụ đích khi image Docker được khởi động. Nó dùng lệnh nc để lắng nghe các cổng của mọi phụ thuộc dịch vụ then chốt mà licensing service cần, rồi chặn cho đến khi các phụ thuộc đó đã khởi động.

Listing sau đây cho thấy cách run.sh được dùng để khởi chạy licensing service.
