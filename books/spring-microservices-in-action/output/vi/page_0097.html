Nào, lên tàu thôi — release train

Spring Cloud dùng cơ chế phi truyền thống để gắn nhãn các dự án Maven. Spring Cloud là một tập hợp các subproject độc lập. Đội ngũ Spring Cloud phát hành qua cơ chế mà họ gọi là “release train.” Tất cả subproject tạo nên Spring Cloud được đóng gói trong một Maven bill of materials (BOM) và phát hành như một khối thống nhất. Đội ngũ Spring Cloud đã dùng tên các ga tàu điện ngầm London làm tên cho các bản phát hành, mỗi bản major tiếp theo lấy tên ga có chữ cái cao hơn. Đã có ba bản phát hành: Angel, Brixton và Camden. Camden là bản mới nhất, nhưng vẫn có nhiều nhánh release candidate cho các subproject bên trong.

Một điểm cần lưu ý là Spring Boot được phát hành độc lập với release train của Spring Cloud. Do đó, các phiên bản Spring Boot khác nhau không tương thích với các bản phát hành Spring Cloud khác nhau. Bạn có thể xem phụ thuộc phiên bản giữa Spring Boot và Spring Cloud, cùng các phiên bản subproject khác nhau trong release train, tại website Spring Cloud (http://projects.spring.io/spring-cloud/).

Bạn vẫn cần thiết lập thêm một file nữa để khởi chạy configuration server lõi. File này là file application.yml nằm trong thư mục confsvr/src/main/resources. File application.yml sẽ cho dịch vụ Spring Cloud configuration biết port nào cần lắng nghe và vị trí back end sẽ cung cấp dữ liệu cấu hình.

Bạn sắp sẵn sàng khởi chạy dịch vụ Spring Cloud configuration. Bạn cần trỏ server tới một repository back-end sẽ chứa dữ liệu cấu hình. Trong chương này, bạn sẽ dùng licensing service mà bạn đã bắt đầu xây dựng ở chương 2 làm ví dụ cách dùng Spring Cloud Config. Để đơn giản, bạn sẽ thiết lập dữ liệu cấu hình ứng dụng cho ba môi trường: môi trường mặc định khi chạy dịch vụ cục bộ, môi trường dev và môi trường production.

Trong Spring Cloud configuration, mọi thứ hoạt động theo một hệ phân cấp. Cấu hình ứng dụng được biểu diễn bằng tên ứng dụng, rồi đến file thuộc tính cho từng môi trường bạn muốn có thông tin cấu hình. Trong mỗi môi trường này, bạn sẽ thiết lập hai thuộc tính cấu hình:

  • Một thuộc tính ví dụ sẽ được licensing service dùng trực tiếp
  • Cấu hình database cho Postgres database bạn sẽ dùng để lưu dữ liệu licensing service
