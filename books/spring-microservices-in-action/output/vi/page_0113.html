88 CHƯƠNG 3 Kiểm soát cấu hình với Spring Cloud configuration server

Cho Spring Cloud Config biết đường dẫn trong Git cần tìm các tệp cấu hình

Ba phần cấu hình chính trong ví dụ trước là các thuộc tính spring.cloud.config.server, spring.cloud.config.server.git.uri và spring.cloud.config.server.git.searchPaths.

Thuộc tính spring.cloud.config.server yêu cầu máy chủ cấu hình Spring Cloud dùng kho back end không dựa trên hệ thống tệp. Trong ví dụ trước, bạn sẽ kết nối tới kho Git trên cloud, GitHub.

Các thuộc tính spring.cloud.config.server.git.uri cung cấp URL của kho lưu trữ mà bạn đang kết nối tới. Cuối cùng, thuộc tính spring.cloud.config.server.git.searchPaths cho máy chủ Spring Cloud Config biết các đường dẫn tương đối trên kho Git cần được tìm kiếm khi máy chủ cấu hình Cloud khởi động. Giống như phiên bản cấu hình dùng hệ thống tệp, giá trị trong thuộc tính spring.cloud.config.server.git.seachPaths sẽ là danh sách phân tách bằng dấu phẩy cho từng dịch vụ được máy chủ cấu hình lưu trữ.

3.3.6 Làm mới các thuộc tính của bạn bằng máy chủ cấu hình Spring Cloud

Một trong những câu hỏi đầu tiên mà các nhóm phát triển thường đặt ra khi muốn dùng máy chủ cấu hình Spring Cloud là làm thế nào để làm mới động ứng dụng khi một thuộc tính thay đổi. Máy chủ cấu hình Spring Cloud sẽ luôn phục vụ phiên bản mới nhất của một thuộc tính. Các thay đổi được thực hiện đối với thuộc tính qua kho lưu trữ bên dưới sẽ luôn được cập nhật.

Tuy nhiên, các ứng dụng Spring Boot chỉ đọc thuộc tính của chúng tại thời điểm khởi động, nên các thay đổi thuộc tính trên máy chủ cấu hình Spring Cloud sẽ không tự động được ứng dụng Spring Boot nhận. Spring Boot Actuator cung cấp annotation @RefreshScope cho phép nhóm phát triển truy cập endpoint /refresh để buộc ứng dụng Spring Boot đọc lại cấu hình ứng dụng. Listing sau đây trình bày annotation @RefreshScope trong thực tế.
