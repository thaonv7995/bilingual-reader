313 Kích hoạt build dịch vụ trong Travis CI
  1. Thông báo tài khoản GitHub. Travis CI sẽ khởi động máy ảo dùng để thực thi build. Travis CI sau đó checkout mã nguồn từ GitHub rồi dùng tệp .travis.yml để bắt đầu toàn bộ quy trình build và triển khai.
  2. Travis CI thiết lập cấu hình cơ bản trong build và cài đặt các dependency. Cấu hình cơ bản bao gồm ngôn ngữ sẽ dùng trong build (Java), liệu có cần Sudo để cài đặt phần mềm và truy cập Docker (để tạo và gắn tag container Docker), đặt các biến môi trường bảo mật cần thiết trong build, và định nghĩa cách bạn được thông báo khi build thành công hay thất bại.
  3. Trước khi build thực sự được thực thi, Travis CI có thể được hướng dẫn cài đặt bất kỳ thư viện bên thứ ba hoặc công cụ dòng lệnh nào có thể cần trong quy trình build. Bạn dùng hai công cụ như vậy: công cụ dòng lệnh travis và Amazon ecs-cli (EC2 Container Service client).
  4. Đối với quy trình build, luôn bắt đầu bằng gắn tag mã trong kho mã nguồn để bất cứ lúc nào trong tương lai bạn có thể lấy ra phiên bản đầy đủ mã nguồn dựa trên tag của build.
  5. Quy trình build sau đó sẽ thực thi các script Maven cho các dịch vụ. Các script Maven sẽ biên dịch microservice Spring, chạy kiểm thử đơn vị và tích hợp, rồi build image Docker dựa trên build.
  6. Khi image Docker cho build hoàn tất, quy trình build sẽ đẩy image lên Docker Hub với cùng tên tag bạn đã dùng để gắn tag kho mã nguồn.
