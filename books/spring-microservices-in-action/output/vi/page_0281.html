8.4.2 Định nghĩa kênh tùy chỉnh

Trước đó bạn đã xây dựng tích hợp messaging giữa licensing service và organization service để dùng các kênh output và input mặc định đi kèm với các interface Source và Sink trong dự án Spring Cloud Stream. Tuy nhiên, nếu bạn muốn định nghĩa nhiều hơn một kênh cho ứng dụng hoặc muốn tùy chỉnh tên kênh, bạn có thể định nghĩa interface riêng và expose bao nhiêu kênh input và output mà ứng dụng cần.

Để tạo kênh tùy chỉnh, hãy gọi inboundOrgChanges trong licensing service. Bạn có thể định nghĩa kênh trong interface licensing-service/src/main/java/com/thoughtmechanix/licenses/events/CustomChannels.java, như thể hiện trong listing sau.

Điểm chính cần rút ra từ listing 8.12 là: với mỗi kênh input tùy chỉnh bạn muốn expose, bạn đánh dấu một phương thức trả về lớp SubscribableChannel bằng annotation @Input. Nếu bạn muốn định nghĩa kênh output để publish message, bạn sẽ dùng @OutputChannel phía trên phương thức sẽ được gọi. Trong trường hợp kênh output, phương thức được định nghĩa sẽ trả về lớp MessageChannel thay vì lớp SubscribableChannel dùng với kênh input:

Giờ bạn đã có kênh input tùy chỉnh được định nghĩa, bạn cần sửa đổi thêm hai điều nữa để dùng nó trong licensing service. Đầu tiên, bạn cần sửa licensing service để ánh xạ tên kênh input tùy chỉnh tới Kafka topic. Listing sau trình bày điều này.
