9.2.5 Thêm correlation ID vào HTTP response với Zuul

Nếu bạn kiểm tra HTTP response trả về từ bất kỳ lời gọi dịch vụ nào thực hiện với Spring Cloud Sleuth, bạn sẽ thấy trace ID được dùng trong lời gọi không bao giờ được trả về trong HTTP response header. Nếu bạn xem tài liệu của Spring Cloud Sleuth, bạn sẽ thấy nhóm Spring Cloud Sleuth cho rằng việc trả về bất kỳ dữ liệu tracing nào cũng có thể là vấn đề bảo mật tiềm ẩn (mặc dù họ không nêu rõ lý do tại sao họ cho như vậy.)

Tuy nhiên, tôi thấy việc trả về correlation ID hoặc tracing ID trong HTTP response vô cùng hữu ích khi debug sự cố. Spring Cloud Sleuth cho phép bạn “decorate” thông tin HTTP response bằng tracing ID và span ID của nó. Tuy nhiên, quy trình làm việc này đòi hỏi viết ba class và inject hai Spring bean tùy chỉnh. Nếu bạn muốn dùng cách này, bạn có thể xem trong tài liệu Spring Cloud Sleuth (http://cloud.spring.io/spring-cloud-static/spring-cloud-sleuth/1.0.12.RELEASE/). Một giải pháp đơn giản hơn nhiều là viết một Zuul “POST” filter để inject trace ID vào HTTP response.

Ở chương 6 khi giới thiệu Zuul API gateway, chúng ta đã xem cách xây dựng Zuul “POST” response filter để thêm correlation ID mà bạn tạo ra để dùng trong các dịch vụ vào HTTP response mà caller nhận được. Giờ bạn sẽ sửa filter đó để thêm Spring Cloud Sleuth header.

Để thiết lập Zuul response filter, bạn cần thêm một dependency JAR vào file pom.xml của Zuul server: spring-cloud-starter-sleuth. Dependency spring-cloud-starter-sleuth sẽ được dùng để báo cho Spring Cloud Sleuth rằng bạn muốn Zuul tham gia vào Spring Cloud trace. Về sau trong chương, khi giới thiệu Zipkin, bạn sẽ thấy dịch vụ Zuul sẽ là lời gọi đầu tiên trong mọi service invocation.

Cho chương 9, file này có thể tìm thấy tại zuulsvr/pom.xml. Listing sau đây trình bày các dependency này.

Sau khi dependency mới này đã được thêm vào, chính Zuul “post” filter thực tế rất dễ triển khai. Listing sau đây trình bày mã nguồn dùng để xây dựng Zuul filter. File nằm tại zuulsvr/src/main/java/com/thoughtmechanix/zuulsvr/filters/ResponseFilter.java.
