8.4.1 Sử dụng Redis để lưu trữ bộ nhớ đệm cho các tra cứu

Giờ bạn sẽ bắt đầu bằng cách thiết lập licensing service để sử dụng Redis. May mắn là Spring Data đã làm cho việc đưa Redis vào licensing service trở nên đơn giản. Để sử dụng Redis trong licensing service, bạn cần thực hiện bốn việc:

  1. Cấu hình licensing service để bao gồm các dependency Spring Data Redis
  2. Thiết lập kết nối cơ sở dữ liệu tới Redis
  3. Định nghĩa các Spring Data Redis Repository mà code của bạn sẽ dùng để tương tác với một Redis hash
  4. Dùng Redis và licensing service để lưu trữ và đọc dữ liệu tổ chức

Cấu hình licensing service với các dependency Spring Data Redis

Việc đầu tiên bạn cần làm là thêm các dependency spring-data-redis, cùng với các dependency jedis và commons-pool2, vào file pom.xml của licensing service. Các dependency cần thêm được trình bày trong listing sau.

Thiết lập kết nối cơ sở dữ liệu tới máy chủ Redis

Giờ bạn đã có các dependency trong Maven, bạn cần thiết lập kết nối tới máy chủ Redis. Spring dùng dự án mã nguồn mở Jedis (https://github.com/xetorthio/jedis) để giao tiếp với máy chủ Redis. Để giao tiếp với một instance Redis cụ thể, bạn sẽ expose một JedisConnectionFactory trong class licensing-service/src/main/java/com/thoughtmechanix/licenses/Application.java như một Spring Bean. Sau khi có kết nối tới Redis, bạn sẽ dùng kết nối đó để tạo đối tượng RedisTemplate của Spring. Đối tượng RedisTemplate sẽ được các class repository Spring Data mà bạn sẽ triển khai ngay sau đây dùng để thực hiện các truy vấn và lưu dữ liệu organization service vào dịch vụ Redis của bạn.
