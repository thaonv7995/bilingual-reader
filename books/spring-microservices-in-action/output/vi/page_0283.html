258 CHƯƠNG 8 Kiến trúc hướng sự kiện với Spring Cloud Stream

Khi nhận được message, hãy kiểm tra hành động đã thực hiện với dữ liệu, rồi phản ứng tương ứng.

Nếu dữ liệu organization được cập nhật hoặc xóa, hãy vô hiệu hóa dữ liệu organization khỏi Redis thông qua lớp OrganizationRedisRepository.

8.5 Tóm tắt

  • Giao tiếp bất đồng bộ bằng messaging là phần then chốt của kiến trúc microservice.
  • Dùng messaging trong ứng dụng giúp các dịch vụ của bạn mở rộng quy mô và trở nên chịu lỗi hơn.
  • Spring Cloud Stream đơn giản hóa việc tạo và tiêu thụ message bằng cách dùng các annotation đơn giản và trừu tượng hóa các chi tiết đặc thù nền tảng của message platform bên dưới.
  • Spring Cloud Stream message source là một phương thức Java có annotation dùng để publish message tới queue của message broker.
  • Spring Cloud Stream message sink là một phương thức Java có annotation nhận message từ queue của message broker.
  • Redis là kho lưu trữ key-value có thể dùng vừa làm cơ sở dữ liệu vừa làm bộ nhớ đệm.
