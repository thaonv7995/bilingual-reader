Một trường hợp sử dụng Spring Cloud Stream: bộ nhớ đệm phân tán 251

Phần công việc nền tảng để thiết lập licensing service giao tiếp với Redis đã hoàn tất. Giờ hãy chuyển sang viết logic sẽ lấy, thêm, cập nhật và xóa dữ liệu từ Redis.

Định nghĩa các repository Spring Data Redis

Redis là một kho lưu trữ key-value hoạt động như một HashMap phân tán, lưu trong bộ nhớ, quy mô lớn. Trong trường hợp đơn giản nhất, nó lưu trữ dữ liệu và tra cứu dữ liệu theo một key. Nó không có ngôn ngữ truy vấn phức tạp nào để truy xuất dữ liệu. Sự đơn giản chính là điểm mạnh của nó và là một trong những lý do khiến rất nhiều dự án đã áp dụng nó trong các dự án của họ.

Vì bạn đang dùng Spring Data để truy cập kho Redis của mình, bạn cần định nghĩa một lớp repository. Như bạn có thể nhớ từ đầu chương 2, Spring Data dùng các lớp repository do người dùng định nghĩa để cung cấp một cơ chế đơn giản cho một lớp Java truy cập cơ sở dữ liệu Postgres của bạn mà không cần viết các truy vấn SQL cấp thấp.

Đối với licensing service, bạn sẽ định nghĩa hai file cho repository Redis của mình. File đầu tiên bạn sẽ viết sẽ là một interface Java sẽ được inject vào bất kỳ
