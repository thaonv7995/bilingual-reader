238 CHƯƠNG 8 Kiến trúc hướng sự kiện với Spring Cloud Stream

Khi xuất bản và tiêu thụ message trong Spring Cloud, có bốn thành phần tham gia vào việc xuất bản và tiêu thụ message:

  • Source
  • Channel
  • Binder
  • Sink

Source

Khi một dịch vụ sẵn sàng xuất bản message, nó sẽ xuất bản message đó bằng source. Source là một interface có annotation Spring nhận một Plain Old Java Object (POJO) đại diện cho message cần xuất bản. Source nhận message, serialize message đó (serialization mặc định là JSON), rồi xuất bản message vào một channel.

Channel

Channel là một abstraction bao bọc queue sẽ chứa message sau khi message được xuất bản bởi message producer hoặc được tiêu thụ bởi message consumer. Tên channel luôn được gắn với tên queue đích. Tuy nhiên, tên queue đó không bao giờ được expose trực tiếp ra code. Thay vào đó, tên channel được dùng trong code, nghĩa là bạn có thể chuyển các queue mà channel đọc hoặc ghi bằng cách thay đổi cấu hình ứng dụng, chứ không phải code ứng dụng.

Binder

Binder là một phần của framework Spring Cloud Stream. Đây là code Spring giao tiếp với một nền tảng message cụ thể. Phần binder trong framework Spring Cloud Stream cho phép bạn làm việc với message mà không phải tiếp xúc với các thư viện và API đặc thù nền tảng dùng để xuất bản và tiêu thụ message.

Sink

Trong Spring Cloud Stream, khi một dịch vụ nhận message từ queue, nó làm điều đó thông qua sink. Sink lắng nghe một channel để nhận message đến và deserialize message trở lại thành plain old Java object. Từ đó, message có thể được xử lý bởi logic nghiệp vụ của dịch vụ Spring.

8.3 Viết message producer và consumer đơn giản

Giờ chúng ta đã đi qua các thành phần cơ bản trong Spring Cloud Stream, hãy xem một ví dụ Spring Cloud Stream đơn giản. Trong ví dụ đầu tiên, bạn sẽ truyền một message từ organization service tới licensing service. Việc duy nhất bạn sẽ làm với message trong licensing service là in một log message ra console.

Ngoài ra, vì trong ví dụ này bạn chỉ có một Spring Cloud Stream source (message producer) và sink (message consumer), bạn sẽ bắt đầu ví dụ với một vài shortcut đơn giản của Spring Cloud giúp việc thiết lập source trong organization service và sink trong licensing service trở nên cực kỳ đơn giản.
