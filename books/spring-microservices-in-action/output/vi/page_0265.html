240 CHƯƠNG 8 Kiến trúc hướng sự kiện với Spring Cloud Stream

Sau khi đã định nghĩa các dependency Maven, bạn cần cho ứng dụng biết rằng nó sẽ bind tới một message broker Spring Cloud Stream. Bạn thực hiện việc này bằng cách gắn annotation @EnableBinding lên bootstrap class của organization service organization-service/src/main/java/com/thoughtmechanix/organization/Application.java. Listing sau đây cho thấy mã nguồn Application.java của organization service.

Trong listing 8.1, annotation @EnableBinding cho Spring Cloud Stream biết rằng bạn muốn bind dịch vụ tới một message broker. Việc dùng Source.class trong annotation @EnableBinding cho Spring Cloud Stream biết rằng dịch vụ này sẽ giao tiếp với message broker thông qua một tập hợp các channel được định nghĩa trên class Source. Hãy nhớ rằng các channel nằm phía trên message queue. Spring Cloud Stream có một tập hợp channel mặc định có thể được cấu hình để giao tiếp với message broker.

Tại thời điểm này, bạn chưa cho Spring Cloud Stream biết message broker nào mà organization service cần bind tới. Chúng ta sẽ làm điều đó ngay sau đây. Giờ, bạn có thể triển khai code sẽ publish một message.
