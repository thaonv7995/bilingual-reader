70 CHƯƠNG 3 Kiểm soát cấu hình với Spring Cloud configuration server
  1. Spring Cloud configuration server cung cấp nhiều back end để lưu trữ dữ liệu cấu hình. Nếu bạn đã dùng các công cụ như Eureka và Consul, bạn có thể tích hợp chúng trực tiếp vào Spring Cloud configuration server.
  2. Trong tất cả các giải pháp ở bảng 3.1, Spring Cloud configuration server có thể tích hợp trực tiếp với nền tảng quản lý mã nguồn Git. Việc tích hợp Git của Spring Cloud configuration loại bỏ một phụ thuộc phụ trong giải pháp của bạn và giúp quản lý phiên bản dữ liệu cấu hình ứng dụng trở nên dễ dàng.

Các công cụ khác (Etcd, Consul, Eureka) không cung cấp tính năng quản lý phiên bản gốc, và nếu bạn muốn có điều đó, bạn sẽ phải tự xây dựng. Nếu công ty bạn dùng Git, việc sử dụng Spring Cloud configuration server là một lựa chọn hấp dẫn.

Trong phần còn lại của chương này, bạn sẽ

  1. Thiết lập Spring Cloud configuration server và trình bày hai cơ chế khác nhau để phục vụ dữ liệu cấu hình ứng dụng—một dùng hệ thống tệp và một dùng kho Git
  2. Tiếp tục mở rộng licensing service để truy xuất dữ liệu từ cơ sở dữ liệu
  3. Tích hợp dịch vụ Spring Cloud configuration vào licensing service để cung cấp dữ liệu cấu hình ứng dụng

3.2 Xây dựng máy chủ cấu hình Spring Cloud

Spring Cloud configuration server là ứng dụng dựa trên REST được xây dựng trên Spring Boot. Nó không đi kèm dưới dạng máy chủ độc lập. Thay vào đó, bạn có thể nhúng nó vào một ứng dụng Spring Boot hiện có hoặc khởi tạo một dự án Spring Boot mới với máy chủ được nhúng sẵn.

Việc đầu tiên bạn cần làm là tạo một thư mục dự án mới có tên confsvr. Trong thư mục consvr, bạn sẽ tạo một tệp Maven mới dùng để tải xuống các JAR cần thiết để khởi động Spring Cloud configuration server. Thay vì đi qua toàn bộ tệp Maven, tôi sẽ liệt kê các phần chính trong bảng liệt kê sau.
