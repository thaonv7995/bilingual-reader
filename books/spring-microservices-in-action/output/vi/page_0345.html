10.6.3 Thực thi build

Đến thời điểm này, toàn bộ cấu hình trước build và cài đặt phụ thuộc đã hoàn tất. Để thực thi build, bạn sẽ dùng thuộc tính script của Travis. Giống như thuộc tính before_install, thuộc tính script nhận một danh sách lệnh sẽ được thực thi. Vì các lệnh này dài, tôi chọn đóng gói từng bước chính trong build vào script shell riêng và để Travis thực thi script shell đó. Listing sau đây trình bày các bước chính sẽ được thực hiện trong build.

Hãy đi qua từng bước chính được thực thi trong bước script.

10.6.4 Gắn thẻ mã nguồn kiểm soát phiên bản

Script travis_scripts/tag_build.sh đảm nhận việc gắn thẻ mã trong kho với tên build. Trong ví dụ này, tôi tạo một bản phát hành GitHub qua GitHub REST API. Một bản phát hành GitHub không chỉ gắn thẻ kho kiểm soát nguồn, mà còn cho phép bạn đăng những thứ như ghi chú phát hành lên trang web GitHub cùng với thông tin liệu mã nguồn có phải là bản pre-release hay không.

Vì GitHub release API là lời gọi dựa trên REST, bạn sẽ dùng curl trong script shell để thực hiện lời gọi thực tế. Listing sau đây trình bày mã từ script travis_scripts/tag_build.sh.
