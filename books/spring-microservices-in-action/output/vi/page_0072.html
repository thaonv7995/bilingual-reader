47 Câu chuyện của lập trình viên: xây dựng microservice với Spring Boot và Java

Báo cho Maven biết cần bao gồm các plugin maven dành riêng cho Spring để build và triển khai ứng dụng Spring Boot

Chúng ta sẽ không đi qua toàn bộ script một cách chi tiết, nhưng hãy lưu ý một vài khu vực then chốt khi bắt đầu.

Spring Boot được chia thành nhiều dự án riêng lẻ. Triết lý là bạn không nên phải “kéo cả thế giới xuống” nếu bạn không dùng các phần khác nhau của Spring Boot trong ứng dụng của mình. Điều này cũng cho phép các dự án Spring Boot khác nhau phát hành phiên bản mã mới độc lập với nhau. Để giúp đơn giản hóa cuộc sống của lập trình viên, đội ngũ Spring Boot đã gom các dự án phụ thuộc liên quan vào các bộ “starter”.

1 Ở phần 1 của Maven POM, bạn báo cho Maven biết cần kéo xuống phiên bản 1.4.4 của framework Spring Boot

2 Ở phần 2 và 3 của tệp Maven, bạn xác định rằng mình đang kéo xuống các bộ starter Spring Web và Spring Actuator. Hai dự án này là cốt lõi của hầu hết mọi dịch vụ Spring Boot dựa trên REST. Bạn sẽ thấy rằng khi xây dựng thêm chức năng cho dịch vụ của mình, danh sách các dự án phụ thuộc này sẽ dài hơn.

4 Ngoài ra, Spring Source đã cung cấp các plugin Maven giúp đơn giản hóa việc build và triển khai ứng dụng Spring Boot. Bước 4 báo cho script build Maven của bạn cài plugin Spring Boot Maven mới nhất. Plugin này chứa một số tác vụ bổ sung (chẳng hạn như spring-boot:run) giúp đơn giản hóa tương tác giữa Maven và Spring Boot.

Cuối cùng, bạn sẽ thấy một comment cho biết các phần của tệp Maven đã bị loại bỏ. Vì môi trường, tôi đã không đưa các plugin Spotify Docker vào listing 2.1.

LƯU Ý Mỗi chương trong cuốn sách này đều có các tệp Docker để build và triển khai ứng dụng dưới dạng container Docker. Bạn có thể tìm chi tiết cách build các Docker image này trong tệp README.md ở phần mã của từng chương.

2.3.2 Khởi động ứng dụng Spring Boot: viết class Bootstrap

Mục tiêu của bạn là đưa một microservice đơn giản lên và chạy trong Spring Boot, rồi phát triển lặp trên đó để cung cấp chức năng. Vì vậy, bạn cần tạo hai class trong microservice licensing service của mình:

  • Một class Spring Bootstrap sẽ được Spring Boot dùng để khởi động và khởi tạo ứng dụng
  • Một class Spring Controller sẽ cung cấp các endpoint HTTP có thể được gọi trên microservice
