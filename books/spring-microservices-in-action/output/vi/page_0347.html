322 CHƯƠNG 10 Triển khai microservice của bạn

Lệnh Maven này thực thi tệp Maven cha spmia-chapter10-code/pom.xml cho tất cả các dịch vụ trong kho mã nguồn chương 10. Tệp pom.xml cha thực thi tệp Maven pom.xml riêng cho từng dịch vụ. Mỗi dịch vụ riêng lẻ build mã nguồn dịch vụ, chạy các bài kiểm thử đơn vị và tích hợp, rồi đóng gói dịch vụ thành một jar có thể thực thi.

Điều cuối cùng xảy ra trong build Maven là tạo image container Docker được đẩy lên kho lưu trữ Docker cục bộ chạy trên máy build Travis của bạn. Việc tạo image Docker được thực hiện bằng plugin Docker của Spotify (https://github.com/spotify/docker-maven-plugin). Nếu bạn quan tâm cách plugin Docker của Spotify hoạt động trong quy trình build, hãy tham khảo phụ lục A, “Thiết lập môi trường desktop của bạn”. Quy trình build Maven và cấu hình Docker được giải thích ở đó.

10.6.6 Đẩy các image lên Docker Hub

Ở thời điểm này trong build, các dịch vụ đã được biên dịch và đóng gói, và một image container Docker đã được tạo trên máy build Travis. Bạn sẽ đẩy image container Docker lên kho lưu trữ Docker trung tâm thông qua script travis_scripts/deploy_to_docker_hub.sh. Kho lưu trữ Docker giống như kho Maven cho các image Docker bạn đã tạo. Các image Docker có thể được gắn tag và tải lên đó, và các dự án khác có thể tải xuống và sử dụng các image.

Trong ví dụ mã này, bạn sẽ dùng Docker Hub (https://hub.docker.com/). Listing sau đây cho thấy các lệnh được dùng trong script travis_scripts/deploy_to_docker_hub.sh.

Luồng xử lý của shell script này rất đơn giản. Việc đầu tiên bạn phải làm là đăng nhập vào Docker Hub bằng công cụ dòng lệnh Docker và thông tin xác thực người dùng của tài khoản Docker Hub mà các image sẽ được đẩy lên.
