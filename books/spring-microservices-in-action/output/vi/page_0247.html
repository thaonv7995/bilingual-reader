Sau khi bạn expose JWTTokenEnhancer dưới dạng bean, bạn có thể gắn nó vào class JWTOAuth2Config từ listing 7.9. Việc này được thực hiện trong phương thức configure() của class. Listing sau đây cho thấy sự sửa đổi đối với phương thức configure() của class JWTOAuth2Config.

Đến lúc này, bạn đã thêm một trường tùy chỉnh vào JWT token của mình. Câu hỏi tiếp theo bạn nên đặt ra là: “Làm thế nào để parse một trường tùy chỉnh ra khỏi JWT token?”

7.4.4 Phân tích trường tùy chỉnh ra khỏi JavaScript token

Chúng ta sẽ quay sang Zuul gateway của bạn để xem một ví dụ về cách parse trường tùy chỉnh trong JWT token. Cụ thể, bạn sẽ sửa đổi class TrackingFilter mà chúng ta đã giới thiệu ở chương 6 để decode trường organizationId ra khỏi JWT token đang đi qua gateway.

Để làm việc này, bạn sẽ kéo vào một thư viện JWT parser và thêm vào file pom.xml của Zuul server. Có nhiều token parser sẵn có và tôi chọn thư viện JJWT
