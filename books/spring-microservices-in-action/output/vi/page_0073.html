48 CHƯƠNG 2 Xây dựng microservice với Spring Boot

Như bạn sẽ thấy ngay sau đây, Spring Boot dùng annotation để đơn giản hóa việc thiết lập và cấu hình dịch vụ. Điều này trở nên rõ ràng khi bạn xem lớp bootstrap trong listing sau. Lớp bootstrap này nằm trong file src/main/java/com/thoughtmechanix/licenses/Application.java.

@SpringBootApplication cho framework Spring Boot biết đây là lớp bootstrap của dự án

Lệnh gọi để khởi động toàn bộ dịch vụ Spring Boot

Điều đầu tiên cần lưu ý trong đoạn code này là việc dùng annotation @SpringBootApplication. Spring Boot dùng annotation này để báo cho Spring container rằng lớp này là nguồn định nghĩa bean dùng trong Spring. Trong ứng dụng Spring Boot, bạn có thể định nghĩa Spring Bean bằng cách

  1. Gắn annotation @Component, @Service hoặc @Repository cho một lớp Java
  2. Gắn annotation @Configuration cho một lớp, sau đó định nghĩa phương thức constructor cho mỗi Spring Bean bạn muốn tạo bằng annotation @Bean.

Bên trong, annotation @SpringBootApplication đánh dấu lớp Application trong listing 2.2 là lớp cấu hình, rồi bắt đầu tự động quét tất cả các lớp trên Java class path để tìm các Spring Bean khác.

Điều thứ hai cần lưu ý là phương thức main() của lớp Application. Trong phương thức main(), lệnh gọi SpringApplication.run(Application.class, args) khởi động Spring container và trả về một đối tượng Spring ApplicationContext. (Bạn không làm gì với ApplicationContext, nên nó không được hiển thị trong code.)

Điều dễ nhớ nhất về annotation @SpringBootApplication và lớp Application tương ứng là đây là lớp bootstrap cho toàn bộ microservice. Logic khởi tạo cốt lõi của dịch vụ nên được đặt trong lớp này.
