328 PHỤ LỤC A Chạy cloud trên desktop của bạn

Vì mục tiêu đó, bạn sẽ thấy các công nghệ và mẫu thiết kế sau được dùng xuyên suốt mọi chương trong cuốn sách này:

  1. Tất cả dự án dùng Apache Maven (http://maven.apache.org) làm công cụ build cho các chương. Mỗi dịch vụ được build theo cấu trúc dự án Maven và cấu trúc dịch vụ được bố trí nhất quán từ chương này sang chương khác.
  2. Tất cả dịch vụ được phát triển trong chương được biên dịch thành image container Docker (http://docker.io). Docker là một engine ảo hóa thời gian chạy tuyệt vời, chạy trên Windows, OS X và Linux. Nhờ Docker, tôi có thể dựng một môi trường thời gian chạy hoàn chỉnh trên desktop bao gồm các dịch vụ ứng dụng và toàn bộ hạ tầng cần thiết để hỗ trợ các dịch vụ đó. Hơn nữa, không giống các công nghệ ảo hóa độc quyền hơn, Docker dễ mang sang nhiều nhà cung cấp cloud.

Tôi dùng plugin Docker Maven của Spotify (https://github.com/spotify/docker-maven-plugin) để tích hợp việc build container Docker vào quy trình build Maven.

  1. Để khởi động các dịch vụ sau khi chúng đã được biên dịch thành image Docker, tôi dùng Docker Compose để khởi động các dịch vụ theo nhóm. Tôi cố ý tránh các công cụ điều phối Docker phức tạp hơn như Kubernetes (https://github.com/kubernetes/kubernetes) hoặc Mesos (http://mesos.apache.org/) để giữ các ví dụ trong chương đơn giản và dễ mang đi.

Mọi việc triển khai các image Docker đều được thực hiện bằng shell script đơn giản.

A.1 Phần mềm bắt buộc

Để build phần mềm cho tất cả các chương, bạn cần cài các phần mềm sau trên desktop. Quan trọng là đây là các phiên bản phần mềm tôi đã dùng khi viết sách. Phần mềm có thể hoạt động với phiên bản khác, nhưng đây là phiên bản tôi đã build mã nguồn:

  1. Apache Maven (http://apache.maven.org)—Tôi dùng Maven phiên bản 3.3.9. Tôi chọn Maven vì dù các công cụ build khác như Gradle rất phổ biến, Maven vẫn là công cụ build chủ đạo trong hệ sinh thái Java. Tất cả ví dụ mã trong sách này được biên dịch bằng Java phiên bản 1.8.
  2. Docker (http://docker.com)—Tôi build các ví dụ mã trong sách này bằng Docker V1.12. Các ví dụ mã trong sách sẽ hoạt động với phiên bản Docker cũ hơn, nhưng bạn có thể phải chuyển sang định dạng docker-compose links phiên bản 1 nếu muốn dùng mã này với phiên bản Docker cũ hơn.
  3. Git Client (http://git-scm.com)—Toàn bộ mã nguồn của sách được lưu trong kho GitHub. Khi viết sách, tôi dùng Git client phiên bản 2.8.4.

Tôi sẽ không hướng dẫn chi tiết cách cài từng thành phần này. Mỗi gói phần mềm trong danh sách đều có hướng dẫn cài đặt đơn giản và nên cài được với ít công sức. Docker có client GUI để cài đặt.
