RestTemplate và UserContextInterceptor tùy chỉnh: Đảm bảo correlation ID được truyền tiếp

Phần mã cuối cùng mà chúng ta sẽ xem là class UserContextInterceptor. Class này được dùng để inject correlation ID vào mọi yêu cầu dịch vụ dựa trên HTTP đi ra được thực thi từ một instance RestTemplate. Việc này nhằm đảm bảo bạn có thể thiết lập liên kết giữa các lời gọi dịch vụ.

Để làm điều này, bạn sẽ dùng một Spring Interceptor được inject vào class RestTemplate. Hãy xem UserContextInterceptor trong listing sau đây.

Để dùng UserContextInterceptor, bạn cần định nghĩa một bean RestTemplate rồi thêm UserContextInterceptor vào đó. Để làm điều này, bạn sẽ thêm định nghĩa bean RestTemplate của riêng mình vào class licensing-service/src/main/java/com/thoughtmechanix/licenses/Application.java. Listing sau đây cho thấy phương thức được thêm vào class này.
