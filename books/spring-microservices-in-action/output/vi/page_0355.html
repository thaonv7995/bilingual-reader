A.3 Cấu trúc từng chương

Mỗi chương trong sách có một hoặc nhiều dịch vụ liên quan. Mỗi dịch vụ trong một chương có thư mục dự án riêng. Ví dụ, nếu bạn xem chương 6 (http://github.com/carnellj/spmia-chapter6), bạn sẽ thấy có bảy dịch vụ. Các dịch vụ này là

  1. confsvr—Spring Cloud Config server
  2. eurekasvr—Spring Cloud/with Eureka
  3. licensing-service—Eagle Eye Licensing service
  4. organization-service—Eagle Organization service
  5. orgservice-new—Phiên bản thử nghiệm mới của dịch vụ EagleEye
  6. specialroutes-service—Dịch vụ định tuyến A/B
  7. zuulsvr—EagleEye Zuul service

Mỗi thư mục dịch vụ trong một chương được cấu trúc như một dự án build dựa trên Maven. Bên trong mỗi dự án có thư mục src/main với các thư mục con sau:

  1. java—Thư mục này chứa mã nguồn Java dùng để build dịch vụ.
  2. docker—Thư mục này chứa hai file cần thiết để build image Docker cho mỗi dịch vụ. File đầu tiên luôn có tên Dockerfile và chứa các hướng dẫn từng bước để Docker build image Docker. File thứ hai, run.sh, là script Bash tùy chỉnh chạy bên trong container Docker. Script này đảm bảo dịch vụ không khởi động cho đến khi các phụ thuộc quan trọng (cơ sở dữ liệu đã sẵn sàng) khả dụng.
  3. resources—Thư mục resources chứa tất cả các file application.yml của các dịch vụ. Trong khi cấu hình ứng dụng được lưu trong Spring Cloud Config, tất cả dịch vụ đều có cấu hình được lưu cục bộ trong application.yml. Ngoài ra, thư mục resources sẽ chứa file schema.sql với tất cả các lệnh SQL dùng để tạo bảng và nạp dữ liệu sẵn cho các dịch vụ vào cơ sở dữ liệu Postgres.
