50 CHƯƠNG 2 Xây dựng microservice với Spring Boot

@RestController báo cho Spring Boot biết đây là dịch vụ dựa trên REST và sẽ tự động serialize/deserialize request/response của dịch vụ sang JSON.

Mở tất cả các HTTP endpoint trong lớp này với tiền tố /v1/organizations/{organizationId}/licenses

Chúng ta sẽ bắt đầu tìm hiểu bằng cách xem xét annotation @RestController. @RestController là annotation cấp lớp trong Java và báo cho Spring Container biết lớp Java này sẽ được dùng cho dịch vụ dựa trên REST. Annotation này tự động xử lý việc serialize dữ liệu truyền vào dịch vụ dưới dạng JSON hoặc XML (mặc định, lớp @RestController sẽ serialize dữ liệu trả về thành JSON). Khác với annotation @Controller truyền thống của Spring, annotation @RestController không yêu cầu bạn — với vai trò nhà phát triển — phải trả về lớp ResponseBody từ lớp controller. Tất cả đều được xử lý nhờ sự hiện diện của annotation @RestController, annotation này bao gồm cả annotation @ResponseBody.

Tại sao dùng JSON cho microservice?

Có thể dùng nhiều giao thức để gửi dữ liệu qua lại giữa các microservice dựa trên HTTP. JSON đã trở thành chuẩn de facto vì một số lý do.

Thứ nhất, so với các giao thức khác như SOAP (Simple Object Access Protocol) dựa trên XML, JSON cực kỳ nhẹ — bạn có thể biểu diễn dữ liệu mà không cần quá nhiều phần văn bản thừa.

Thứ hai, con người dễ đọc và tiêu thụ JSON. Đây là một phẩm chất bị đánh giá thấp khi chọn giao thức serialize. Khi sự cố xảy ra, việc nhà phát triển nhìn vào một khối JSON và nhanh chóng xử lý trực quan nội dung bên trong là rất quan trọng. Sự đơn giản của giao thức khiến việc này trở nên cực kỳ dễ dàng.

Thứ ba, JSON là giao thức serialize mặc định được dùng trong JavaScript. Với sự bùng nổ mạnh mẽ của JavaScript như một ngôn ngữ lập trình và sự trỗi dậy không kém phần ấn tượng của Single Page Internet Applications (SPIA) — những ứng dụng phụ thuộc nhiều vào JavaScript — JSON trở thành lựa chọn tự nhiên để xây dựng ứng dụng dựa trên REST vì đây chính là định dạng mà client web phía front-end dùng để gọi dịch vụ.

Các cơ chế và giao thức khác hiệu quả hơn JSON khi giao tiếp giữa các dịch vụ. Framework Apache Thrift (http://thrift.apache.org) cho phép bạn xây dựng dịch vụ đa ngôn ngữ có thể giao tiếp với nhau bằng giao thức nhị phân. Giao thức Apache Avro (http://avro.apache.org) là giao thức serialize dữ liệu chuyển đổi dữ liệu qua lại sang định dạng nhị phân giữa các lần gọi client và server.

Nếu bạn cần giảm thiểu kích thước dữ liệu gửi qua mạng, tôi khuyên bạn nên xem xét các giao thức này. Nhưng theo kinh nghiệm của tôi, dùng JSON thuần trong microservice hoạt động hiệu quả và không chèn thêm một lớp giao tiếp khác cần debug giữa consumer dịch vụ và client dịch vụ.
