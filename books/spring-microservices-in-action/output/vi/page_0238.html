7.4 JavaScript Web Tokens và OAuth2

OAuth2 là một framework xác thực dựa trên token, nhưng trớ trêu thay nó không cung cấp tiêu chuẩn nào về cách định nghĩa các token trong đặc tả của mình. Để khắc phục sự thiếu hụt tiêu chuẩn xung quanh các token OAuth2, một tiêu chuẩn mới đang nổi lên mang tên JavaScript Web Tokens (JWT). JWT là một tiêu chuẩn mở (RFC-7519) do Internet Engineering Task Force (IETF) đề xuất, nhằm cung cấp cấu trúc tiêu chuẩn cho các token OAuth2. Các token JWT

  • Nhỏ gọn—Các token JWT được mã hóa sang Base64 và có thể truyền dễ dàng qua URL, HTTP header hoặc tham số HTTP POST.
  • Được ký mật mã—Một token JWT được máy chủ xác thực phát hành ký. Điều này có nghĩa là bạn có thể chắc chắn token chưa bị can thiệp.
  • Tự chứa—Vì token JWT được ký mật mã, microservice nhận dịch vụ có thể chắc chắn nội dung token là hợp lệ. Không cần gọi lại dịch vụ xác thực để xác thực nội dung token vì chữ ký của token có thể được xác thực và
