10 CHƯƠNG 1 Chào mừng đến với đám mây, Spring

Báo cho framework Spring Boot biết lớp này là điểm vào cho dịch vụ Spring Boot

Báo cho Spring Boot biết bạn sẽ phơi bày mã trong lớp này dưới dạng lớp Spring RestController

Tất cả URL được phơi bày trong ứng dụng này sẽ có tiền tố /hello.

Spring Boot sẽ phơi bày một điểm cuối REST dạng GET nhận hai tham số: firstName và lastName.

Ánh xạ các tham số firstName và lastName được truyền qua URL vào hai biến truyền vào hàm hello

Trả về một chuỗi JSON đơn giản mà bạn tự xây dựng. Ở chương 2, bạn sẽ không tạo JSON nào cả.

Trong bảng liệt kê 1.1, bạn về cơ bản đang phơi bày một điểm cuối HTTP GET duy nhất nhận hai tham số (firstName và lastName) trên URL, rồi trả về một chuỗi JSON đơn giản có payload chứa thông điệp “Hello firstName lastName”. Nếu bạn gọi điểm cuối /hello/john/carnell trên dịch vụ (tôi sẽ trình bày ngay sau đây), kết quả trả về của lời gọi sẽ là

{"message":"Hello john carnell"}

Hãy khởi động dịch vụ. Để làm điều này, hãy mở dòng lệnh và chạy lệnh sau:

mvn spring-boot:run

Lệnh này, mvn, sẽ dùng plug-in Spring Boot để khởi động ứng dụng bằng máy chủ Tomcat nhúng.

Java so với Groovy và Maven so với Gradle

Framework Spring Boot hỗ trợ mạnh cả Java và ngôn ngữ lập trình Groovy. Bạn có thể xây dựng microservice bằng Groovy mà không cần thiết lập dự án. Spring Boot cũng hỗ trợ cả Maven và công cụ build Gradle. Tôi đã giới hạn các ví dụ trong cuốn sách này ở Java và Maven. Là người yêu thích Groovy và Gradle từ lâu, tôi có sự tôn trọng lớn đối với ngôn ngữ và công cụ build đó, nhưng để giữ cuốn sách dễ quản lý và tập trung vào nội dung, tôi đã chọn Java và Maven để tiếp cận đối tượng độc giả rộng nhất có thể.
