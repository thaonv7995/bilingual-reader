Đây là endpoint bạn sẽ dùng để kết nối tới cơ sở dữ liệu.

Tại thời điểm này, quá trình tạo cơ sở dữ liệu của bạn sẽ bắt đầu (có thể mất vài phút). Sau khi hoàn tất, bạn cần cấu hình các dịch vụ Eagle Eye để sử dụng cơ sở dữ liệu.

Sau khi cơ sở dữ liệu được tạo (việc này sẽ mất vài phút), bạn sẽ quay lại bảng điều khiển RDS và thấy cơ sở dữ liệu của mình đã được tạo. Hình 10.5 cho thấy màn hình này.

Trong chương này, tôi đã tạo một application profile mới có tên aws-dev cho từng microservice cần truy cập cơ sở dữ liệu PostgreSQL trên Amazon. Tôi đã thêm một application profile máy chủ Spring Cloud Config mới trong kho GitHub Spring Cloud Config (https://github.com/carnellj/config-repo) chứa thông tin kết nối cơ sở dữ liệu Amazon. Các tệp property tuân theo quy ước đặt tên (service-name)-aws-dev.yml trong mỗi tệp property dùng cơ sở dữ liệu mới (licensing service, organization service và authentication service).

Tại thời điểm này, cơ sở dữ liệu của bạn đã sẵn sàng (không tệ khi thiết lập chỉ với khoảng năm cú nhấp chuột). Hãy chuyển sang phần hạ tầng ứng dụng tiếp theo và xem cách tạo cluster Redis mà licensing service EagleEye của bạn sẽ sử dụng.

10.1.2 Tạo cluster Redis trên Amazon

Để thiết lập cluster Redis, bạn sẽ dùng dịch vụ Amazon ElastiCache. Amazon ElastiCache cho phép bạn xây dựng bộ nhớ đệm dữ liệu in-memory bằng Redis hoặc Memcached (https://memcached.org/). Với các dịch vụ EagleEye, bạn sẽ chuyển máy chủ Redis đang chạy trong Docker sang ElastiCache.
