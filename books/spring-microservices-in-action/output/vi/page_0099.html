74 CHƯƠNG 3 Kiểm soát cấu hình với Spring Cloud configuration server

Suy nghĩ trước khi triển khai

Tôi khuyên không nên dùng giải pháp dựa trên filesystem cho các ứng dụng cloud quy mô trung bình đến lớn. Cách tiếp cận filesystem có nghĩa là bạn phải triển khai một mount point tệp dùng chung cho tất cả máy chủ cấu hình cloud muốn truy cập dữ liệu cấu hình ứng dụng. Thiết lập máy chủ filesystem dùng chung trên cloud là khả thi, nhưng trách nhiệm duy trì môi trường này thuộc về bạn.

Tôi trình bày cách tiếp cận filesystem như ví dụ dễ nhất để làm quen với Spring Cloud configuration server. Ở phần sau, tôi sẽ hướng dẫn cách cấu hình Spring Cloud configuration server để dùng nhà cung cấp Git trên cloud như Bitbucket hoặc GitHub để lưu trữ cấu hình ứng dụng của bạn.

3.2.1 Thiết lập class Bootstrap Spring Cloud Config

Mọi dịch vụ Spring Cloud được đề cập trong cuốn sách này đều cần một class bootstrap dùng để khởi chạy dịch vụ. Class bootstrap này gồm hai thành phần: một phương thức main() Java đóng vai trò điểm vào để khởi động dịch vụ, và một tập annotation Spring Cloud cho dịch vụ biết loại hành vi Spring Cloud nào sẽ được kích hoạt khi khởi động.

Listing sau đây trình bày lớp confsvr/src/main/java/com/thoughtmechanix/confsvr/Application.java được dùng làm class bootstrap cho dịch vụ cấu hình của bạn.

Phương thức main khởi chạy dịch vụ và khởi động Spring container.

Dịch vụ Spring Cloud Config của bạn là một Spring Boot Application, nên bạn đánh dấu nó bằng @SpringBootApplication.

Annotation @EnableConfigServer bật dịch vụ như một dịch vụ Spring Cloud Config.
