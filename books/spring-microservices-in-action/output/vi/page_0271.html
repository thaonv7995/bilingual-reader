246 CHƯƠNG 8 Kiến trúc hướng sự kiện với Spring Cloud Stream

Vì licensing service là consumer của message, bạn sẽ truyền giá trị Sink.class cho annotation @EnableBinding. Điều này cho Spring Cloud Stream biết cần bind tới message broker bằng Spring Sink interface mặc định. Tương tự Spring Cloud Stream Source interface đã mô tả ở mục 8.3.1, Spring Cloud Stream expose một channel mặc định trên Sink interface. Channel trên Sink interface có tên input và dùng để lắng nghe message đến trên một channel.

Sau khi đã định nghĩa rằng bạn muốn lắng nghe message qua annotation @EnableBinding, bạn có thể viết code để xử lý message nhận từ Sink input channel. Để làm việc này, hãy dùng annotation @StreamListener của Spring Cloud Stream.

Annotation @StreamListener cho Spring Cloud Stream biết cần thực thi phương thức loggerSink() mỗi khi nhận được message từ input channel. Spring Cloud Stream sẽ tự động deserialize message từ channel thành Java POJO có tên OrganizationChangeModel.

Một lần nữa, việc ánh xạ thực tế từ topic của message broker tới input channel được thực hiện trong cấu hình của licensing service. Đối với licensing service, cấu hình được minh họa trong listing sau và có thể tìm thấy trong file licensing-service/src/main/resources/application.yml của licensing service.

Cấu hình trong listing này trông giống cấu hình của organization service. Tuy nhiên, có hai khác biệt quan trọng. Thứ nhất, giờ bạn có channel tên input được định nghĩa dưới thuộc tính spring.cloud.stream.bindings. Giá trị này ánh xạ tới channel Sink.INPUT được định nghĩa trong code từ listing 8.5.
