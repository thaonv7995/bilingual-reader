3.3.5 Sử dụng Spring Cloud configuration server với Git

Như đã đề cập trước đó, việc dùng hệ thống tệp làm kho back end cho Spring Cloud configuration server có thể không thực tế đối với ứng dụng trên cloud vì nhóm phát triển phải thiết lập và quản lý một hệ thống tệp dùng chung được gắn trên tất cả các instance của máy chủ cấu hình Cloud.

Spring Cloud configuration server tích hợp với các kho back end khác nhau có thể dùng để lưu trữ thuộc tính cấu hình ứng dụng. Một cách mà tôi đã dùng thành công là dùng Spring Cloud configuration server với kho quản lý mã nguồn Git.

Bằng cách dùng Git, bạn có thể tận dụng mọi lợi ích của việc đưa các thuộc tính quản lý cấu hình vào kiểm soát mã nguồn và cung cấp cơ chế dễ dàng để tích hợp việc triển khai các tệp cấu hình thuộc tính vào pipeline build và triển khai.

Để dùng Git, bạn thay cấu hình back end hệ thống tệp trong tệp bootstrap.yml của dịch vụ cấu hình bằng cấu hình trong listing sau.
