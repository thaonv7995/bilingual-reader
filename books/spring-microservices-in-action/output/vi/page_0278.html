Một trường hợp sử dụng Spring Cloud Stream: bộ nhớ đệm phân tán 253

Mọi tương tác với Redis đều sẽ với một đối tượng Organization duy nhất được lưu theo khóa của nó.

OrganizationRedisRepositoryImpl chứa toàn bộ logic CRUD (Create, Read, Update, Delete) dùng để lưu trữ và truy xuất dữ liệu từ Redis. Có hai điểm chính cần lưu ý từ code trong listing 8.10:

  • Mọi dữ liệu trong Redis đều được lưu trữ và truy xuất bằng một khóa. Vì bạn đang lưu dữ liệu lấy từ organization service, ID organization là lựa chọn tự nhiên cho khóa dùng để lưu bản ghi tổ chức.
  • Điểm thứ hai cần lưu ý là một máy chủ Redis có thể chứa nhiều hash và cấu trúc dữ liệu bên trong. Trong mọi thao tác với máy chủ Redis, bạn cần cho Redis biết tên của cấu trúc dữ liệu mà bạn đang thực hiện thao tác. Trong listing 8.10, tên cấu trúc dữ liệu bạn đang dùng được lưu trong hằng số HASH_NAME và có tên là “organization.”

Dùng Redis và licensing service để lưu trữ và đọc dữ liệu tổ chức

Giờ bạn đã có code để thực hiện các thao tác với Redis, bạn có thể chỉnh sửa licensing service sao cho mỗi khi licensing service cần dữ liệu tổ chức, nó sẽ kiểm tra bộ nhớ đệm Redis trước khi gọi ra organization service.

Logic kiểm tra Redis sẽ nằm trong class licensing-service/src/main/java/com/thoughtmechanix/licenses/clients/OrganizationRestTemplateClient.java. Code của class này được trình bày trong listing sau đây.
