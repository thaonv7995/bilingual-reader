Hãy lưu ý vài điều về annotation @RefreshScope. Đầu tiên, annotation này chỉ tải lại các thuộc tính Spring tùy chỉnh mà bạn có trong cấu hình ứng dụng. Các hạng mục như cấu hình cơ sở dữ liệu được Spring Data sử dụng sẽ không được tải lại bởi annotation @RefreshScope. Để thực hiện refresh, bạn có thể gọi endpoint http://<yourserver>:8080/refresh.

Về làm mới microservice

Khi dùng dịch vụ Spring Cloud configuration với các microservice, một điều bạn cần cân nhắc trước khi thay đổi thuộc tính một cách động là bạn có thể có nhiều instance của cùng một dịch vụ đang chạy, và bạn sẽ cần refresh tất cả các dịch vụ đó với cấu hình ứng dụng mới. Có nhiều cách để xử lý vấn đề này:

Dịch vụ Spring Cloud configuration có cơ chế dựa trên “push” tên là Spring Cloud Bus, cho phép máy chủ cấu hình Spring Cloud phát thông báo tới tất cả các client đang dùng dịch vụ rằng đã có thay đổi. Spring Cloud configuration yêu cầu một thành phần middleware bổ sung đang chạy (RabbitMQ). Đây là cách cực kỳ hữu ích để phát hiện thay đổi, nhưng không phải tất cả back end cấu hình Spring Cloud đều hỗ trợ cơ chế “push” (tức là máy chủ Consul).

Ở chương tiếp theo, bạn sẽ dùng Spring Service Discovery và Eureka để đăng ký tất cả các instance của một dịch vụ. Một kỹ thuật tôi đã dùng để xử lý sự kiện refresh cấu hình ứng dụng là refresh các thuộc tính ứng dụng trong Spring Cloud configuration, rồi viết một script đơn giản để truy vấn engine service discovery nhằm tìm tất cả các instance của một dịch vụ và gọi trực tiếp endpoint /refresh.

Cuối cùng, bạn có thể khởi động lại tất cả các server hoặc container để nhận thuộc tính mới. Đây là việc khá đơn giản, đặc biệt nếu bạn chạy các dịch vụ trong dịch vụ container như Docker. Khởi động lại container Docker chỉ mất vài giây và sẽ buộc đọc lại cấu hình ứng dụng.

Hãy nhớ rằng các server dựa trên cloud là tạm thời (ephemeral). Đừng ngại khởi chạy các instance mới của dịch vụ với cấu hình mới, chuyển traffic sang các dịch vụ mới, rồi gỡ bỏ các instance cũ.

3.4 Bảo vệ thông tin cấu hình nhạy cảm

Theo mặc định, máy chủ cấu hình Spring Cloud lưu tất cả thuộc tính dưới dạng văn bản thuần (plain text) trong các tệp cấu hình ứng dụng. Điều này bao gồm thông tin nhạy cảm như thông tin đăng nhập cơ sở dữ liệu.

Việc lưu thông tin đăng nhập nhạy cảm dưới dạng plain text trong kho mã nguồn là một thực hành cực kỳ tồi. Thật không may, điều đó xảy ra thường xuyên hơn bạn nghĩ. Spring Cloud Config cho phép bạn mã hóa các thuộc tính nhạy cảm một cách dễ dàng. Spring Cloud Config hỗ trợ cả mã hóa đối xứng (shared secret) và mã hóa bất đối xứng (khóa public/private).
