Theo dõi phân tán với Open Zipkin 275

Thiết lập Spring Cloud Sleuth và Zipkin gồm bốn hoạt động:

  • Thêm file JAR Spring Cloud Sleuth và Zipkin vào các dịch vụ thu thập dữ liệu trace
  • Cấu hình thuộc tính Spring trong mỗi dịch vụ để trỏ tới máy chủ Zipkin sẽ thu thập dữ liệu trace
  • Cài đặt và cấu hình máy chủ Zipkin để thu thập dữ liệu
  • Xác định chiến lược lấy mẫu mà mỗi client sẽ dùng để gửi thông tin tracing tới Zipkin

9.3.1 Thiết lập các dependency Spring Cloud Sleuth và Zipkin

Cho đến giờ bạn đã thêm hai bộ dependency Maven vào các dịch vụ Zuul, licensing và organization. Các file JAR này là dependency spring-cloud-starter-sleuth và spring-cloud-sleuth-core. Các dependency spring-cloud-starter-sleuth được dùng để đưa vào các thư viện Spring Cloud Sleuth cơ bản cần thiết để bật Spring Cloud Sleuth trong một dịch vụ. Các dependency spring-cloud-sleuth-core được dùng khi bạn phải tương tác theo chương trình với Spring Cloud Sleuth (việc này bạn sẽ làm lại ở phần sau trong chương).

Để tích hợp với Zipkin, bạn cần thêm dependency Maven thứ hai tên là spring-cloud-sleuth-zipkin. Listing sau đây trình bày các mục Maven cần có trong các dịch vụ Zuul, licensing và organization sau khi dependency spring-cloud-sleuth-zipkin được thêm vào.

9.3.2 Cấu hình các dịch vụ để trỏ tới Zipkin

Khi đã có các file JAR, bạn cần cấu hình mỗi dịch vụ muốn giao tiếp với Zipkin. Bạn làm điều này bằng cách đặt thuộc tính Spring xác định URL dùng để giao tiếp với Zipkin. Thuộc tính cần đặt là spring.zipkin.baseUrl. Thuộc tính này được đặt trong file thuộc tính application.yml của mỗi dịch vụ.

LƯU Ý

spring.zipkin.baseUrl cũng có thể được externalize thành thuộc tính trong Spring Cloud Config.

Trong file application.yml của mỗi dịch vụ, giá trị được đặt là http://localhost:9411. Tuy nhiên, lúc chạy tôi ghi đè giá trị này bằng ZIPKIN_URI
