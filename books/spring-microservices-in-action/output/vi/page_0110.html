khi khởi động sẽ phân tích tên các phương thức từ interface Repository, chuyển đổi chúng thành câu lệnh SQL dựa trên tên phương thức, rồi sinh một lớp proxy động bên dưới để thực hiện công việc. Repository cho licensing service được trình bày trong listing sau.

Báo cho Spring Boot biết đây là một lớp repository JPA

Xác định rằng bạn đang mở rộng Spring CrudRepository

Các phương thức truy vấn riêng lẻ được Spring phân tích thành truy vấn SELECT…FROM.

Interface repository, LicenseRepository, được đánh dấu bằng annotation @Repository, báo cho Spring biết rằng nó nên coi interface này là một repository và sinh một proxy động cho nó. Spring cung cấp nhiều loại repository khác nhau để truy cập dữ liệu. Bạn đã chọn dùng lớp cơ sở Spring CrudRepository để mở rộng lớp LicenseRepository của mình. Lớp cơ sở CrudRepository chứa các phương thức CRUD cơ bản. Ngoài phương thức CRUD được kế thừa từ CrudRepository, bạn đã thêm hai phương thức truy vấn tùy chỉnh để lấy dữ liệu từ bảng licensing. Spring Data framework sẽ phân tích tên các phương thức để xây dựng truy vấn truy cập dữ liệu bên dưới.

LƯU Ý

Spring Data framework cung cấp một lớp trừu tượng trên nhiều nền tảng cơ sở dữ liệu khác nhau và không giới hạn ở cơ sở dữ liệu quan hệ. Các cơ sở dữ liệu NoSQL như MongoDB và Cassandra cũng được hỗ trợ.

Khác với phiên bản trước của licensing service ở chương 2, giờ bạn đã tách logic nghiệp vụ và logic truy cập dữ liệu cho licensing service ra khỏi LicenseController và đưa vào một lớp Service độc lập tên là LicenseService.
