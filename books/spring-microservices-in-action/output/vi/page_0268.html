Thuộc tính cấu hình spring.stream.bindings.output trong listing ánh xạ channel source.output() trong listing 8.2 tới orgChangeTopic trên message broker mà bạn sẽ giao tiếp. Nó cũng cho Spring Cloud Stream biết rằng các message gửi tới topic này nên được serialize dưới dạng JSON. Spring Cloud Stream có thể serialize message ở nhiều định dạng, bao gồm JSON, XML và định dạng Avro của Apache Foundation (https://avro.apache.org/).

Thuộc tính cấu hình spring.stream.bindings.kafka trong listing 8.3 cũng cho Spring Cloud Stream biết cần bind dịch vụ tới Kafka. Các thuộc tính con cho Spring Cloud Stream biết địa chỉ mạng của các message broker Kafka và các máy chủ Apache ZooKeeper chạy cùng Kafka.

Giờ bạn đã viết xong code để publish message qua Spring Cloud Stream và cấu hình cho Spring Cloud Stream biết sẽ dùng Kafka làm message broker, hãy xem việc publish message trong organization service thực sự diễn ra ở đâu. Công việc này sẽ được thực hiện trong class organization-service/src/main/java/com/thoughtmechanix/organization/services/OrganizationService.java. Listing sau đây trình bày code của class này.
