postgresql.conf

postgresql.conf controls the life-sustaining settings of the PostgreSQL server instance as well as default settings for new databases. You can override many settings at the database, user, session, and even function levels. You’ll find many details on how to fine-tune your server by tweaking settings in the article Tuning Your PostgreSQL Server.

An easy way to check the current settings is to query the pg_settings view, as we demonstrate in Example 2-2. We provide a synopsis of key setting and description of the key columns, but to delve deeper, we suggest you check the official documentation, pg_settings.

(1) If context is set to postmaster, changing this parameter requires a restart of the PostgreSQL service. If it’s set to user, changes just require a reload to take effect globally. Restarting terminates active connections, whereas reloading does not.
