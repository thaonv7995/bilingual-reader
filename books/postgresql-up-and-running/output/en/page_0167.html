Prior to version 9.2, SQL functions could use only the ordinal posi­tion of the input arguments in the body of the function. From ver­sion 9.2 onward, you have the option of using named arguments. For example, you can write param_1, param_2 instead of $1, $2. Languag­es other than SQL did not have this limitation before version 9.2.

Functions in almost all languages can return sets. SQL functions are no exception. There are three common approaches to do this: the ANSI SQL standard RETURNS TABLE syntax, OUT parameters, and composite data types. The RETURNS TABLE approach requires at least version 8.3 and is closest to what you’ll find in other database products. In Example 8-4, we demonstrate how to write the same function three ways.
