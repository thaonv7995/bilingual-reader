Now, if this method is not able to find any properties file that matches hi_IN locale (i.e. helloapp_hi_IN.properties or helloapp_hi.properties), it will try to create and return the bundle for the default locale, i.e., helloapp_en_US. If it is not able to find any properties file for en_US locale either, it will return the base bundle, i.e., helloapp. The following image illustrates the search path that getBundle takes in this example:

At this point, you might be curious to know what happens if getBundle is not able to find any properties file that it can use to create any bundle at all. Well, in that case it throws a MissingResourceException.

The exam does not try to trick you on the location of the properties files. Unless the problem statement states otherwise, you can safely assume that the properties file given in the question is on the classpath and can be loaded by a ResourceBundle. You just need to check the name of the properties file and the name of the resource bundle being requested using the getBundle method.
