What is called “exception” (i.e. exception with a lower case ‘e’) in common parlance, is in reality is an object of class java.lang.Throwable or one of its subclasses. Although you will rarely use the Throwable class itself directly in your code, it contains most of the functionality related to exceptions and so, you should be aware of the constructors and important methods of this class.

Throwable constructors☝

Throwable has the following are three public constructors:

1. Throwable() : Constructs a new throwable with null as its detail message.

2. Throwable(String message) : Constructs a new throwable with the specified detail message.

3. Throwable(String message, Throwable cause) : Constructs a new throwable with the specified detail message and the “cause”. The cause is another throwable object that the developer believes is the real cause of the exception. This is useful if you want to catch a standard exception such as NullPointerException and then throw a new business exception with the original exception as the real cause.

Throwable methods☝

You may see the usage of the following methods of Throwable on the exam:

1. String toString() : Returns the string representation of the throwable in <exception class name>: <message> format.

2. String getMessage() : Returns the detail message string of this throwable.

3. void printStackTrace() : Prints the stack trace in the error stream.

4. Throwable getCause() : Returns the cause that was passed while constructing the throwable.

5. void addSuppressed(Throwable exception)/Throwable[] getSuppressed() : Add another throwable to its array of suppressed exceptions and get the array of suppressed exception.
