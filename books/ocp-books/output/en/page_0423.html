19.2.2 Using the Path interface☝

You have already seen file names and paths in the previous section. They were all just strings and they used platform specific separators. The java.nio.file package formally abstracts the concept of a path using the java.nio.file.Path interface. Other classes of this package use Path objects instead of strings whenever they need to refer to a file.

There is no publicly accessible class that implements the Path interface, which means, you cannot instantiate Path objects directly. You need to get Path objects using either an existing Path object or by using the static of(...) methods of the Path interface.
