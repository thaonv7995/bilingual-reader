15.1.4 Returning a value from try, catch, and finally blocks☝

A return statement returns a value and ends the method execution immediately ignoring the rest of the method code, while a finally block prevents the termination of a method before the code in the finally block is executed. Thus, the return statement and the finally block impose contradictory requirements on the flow of execution. This is not a problem when a return statement is present in the try or the catch blocks.
