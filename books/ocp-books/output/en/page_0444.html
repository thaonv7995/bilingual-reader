Although the OCP 21 exam does not go into the implementation details of readObject / writeObject methods, it does expect you to know that their access modifier must be private. So, for example, if a class has a readObject method that is public, it won't be invoked during deserialization of an object of that class and deserialization will occur normally.

Implement Externalizable

The second approach is to implement the java.io.Externalizable interface instead of implementing the Serializable interface. This interface contains two abstract methods that the class must implement:

  1. void readExternal(ObjectInput in)
  2. void writeExternal(ObjectOutput out)

I am not going into the details of these approaches because neither are they required for the exam nor are they used professionally very often. It is good to be aware of these approaches for technical interviews though. You may go through "Java Object Serialization Specification" for more details if you have time.
