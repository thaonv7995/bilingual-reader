Closeable is actually a pre-Java 7 interface in the java.io package and has the same close method. It too was meant to represent a resource that can be closed. However, in Java 7 it was made to extend AutoCloseable so that all classes that implemented Closeable could also be managed using a try-with-resource statement without requiring any change. Thus, option E would have been valid if it had said “implements” instead of “extends”.
