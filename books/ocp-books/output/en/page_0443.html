19.3.3 Customizing object serialization☝

Usually, the readymade serialization logic contained in ObjectInputStream and ObjectOutputStream classes is sufficient to read and write serialized objects. In the rare case that you want to tweak this logic there are a couple of approaches.

Implement writeObject, readObject, and, readObjectNoData methods

In this approach, your class must still implement Serializable but besides that, it may also provide implementation for the following three methods:

  1. private void writeObject(java.io.ObjectOutputStream out) throws IOException
  2. private void readObject(java.io.ObjectInputStream in) throws IOException, ClassNotFoundException
  3. private void readObjectNoData() throws ObjectStreamException
