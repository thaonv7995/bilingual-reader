Observe that when selector is 0, the switch returns 0.1, which is double value, when selector is 1, the switch returns 2, which is an int value, and when selector is neither 0 nor 1, the switch returns ‘x’, which is a char value. So, now, the conundrum is how should the variable value be defined. If you define value as an int, obviously, a double or a char value cannot be assigned to it and the same argument holds if you define it as char. The only acceptable type is double, because a variable of type double can be assigned all three types of values.

Furthermore, since everything is an object in Java, you may also define value as Object. However, by defining value as Object, you lose all of the type information that you will have if you define it as a double. A double is more specific than Object.

For this reason, the type of switch expression is always determined by the compiler to be the most specific type that is compatible with all of the types of the values that the given switch expression can possibly return. It tries to keep as much of type information associated with the return value as possible. The following example illustrates the same point with objects instead of primitives:

Of course, since an Animal is also an Object, you may define the variable pet as Object as well. In strictly technical terms, the type of a switch expression is the most specific upper bound of all the possible types returned by the switch expression. Don’t worry too much about it right now. It will be clear to you after you go through the chapter on Inheritance.
