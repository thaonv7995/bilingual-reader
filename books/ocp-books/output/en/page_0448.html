19.4.1 Using System.out and System.in☝

Writing to the console using System.out

The easiest way to output text to the console is to use java.lang.System class’s public static variable named out. Indeed, we have been using it since our first Java program! The out variable is of type java.io.PrintStream and the PrintStream object pointed to by this variable is connected to the console. That is why whatever we print using its print or println methods, it shows up on the console.

A PrintStream is a FilterOutputStream and is used to “print” representations of various data values conveniently. In addition to the overloaded write methods, which accept a byte[], it has several overloaded print and println methods that accept various primitive data types as well as String and Object. Its printf method is particularly useful for outputting formatted text. You should check out the JavaDoc description of this class for details but let me mention a couple of important point about this class here anyway.

First, even though a PrintStream is able to print all sorts of objects including strings, it is technically a binary stream, which means, it writes binary data to its destination (which is the console in case of System.out). So, whenever you use its printXXX methods, it converts the input into bytes using the given encoding or charset, or the default charset if not specified.

Second, unlike other I/O stream classes, methods of PrintStream do not throw any checked exception, which is why a Java beginner’s “Hello World” program can be written without worrying about exception handling. To check if any error occurred while writing any data to the underlying stream, its checkError() method can be called.

Besides System.out, there is System.err as well, which is also of type PrintStream. This stream is the “standard error stream” and is used by the JVM to print exception stack traces
