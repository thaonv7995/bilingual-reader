Any path that starts with the separator character is assumed to be starting from the root of the file system (on all OSs). Thus, on a Windows machine, if your path starts with a \ and your current directory is on the C drive, then \temp\test.txt will refer to c:\temp\test.txt .

The File class has two more public static fields for separating paths in a platform independent manner. These are:

  1. public static char pathSeparatorChar : It is the system-dependent path-separator character. On windows, it is ; and on Linux, it is : .
  2. public static String pathSeparator : A String version of pathSeparatorChar for convenience. It contains the same character as pathSeparatorChar.

Although completely beyond the scope of the exam, it is good to know that these fields come in handy when you want to dynamically execute a command that expects a list of paths as an argument.
