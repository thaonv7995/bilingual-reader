The following are a few points worth noting about the above example:

  1. The of method provides a way to convert a String into a Path object. The strings used to create Path objects follow the same conventions and syntax as the ones you saw in the previous section. Thus, ./files is a relative path because it doesn't start with a root component. It will be interpreted as being relative to the current directory, which is C:\temp in this case.

  2. The getFileName method returns the farthest (i.e. the last) element of the path, which is files in this case.

  3. The subpath method returns a relative path that is a subsequence of the name elements of this path. The returned Path object has the name elements that begin at beginIndex and extend to the element at index endIndex-1. Note that it returns a relative path, which means, the returned path will never start with the root.

  4. The getRoot method returns the root if it is invoked on an absolute path. Since ./files is not an absolute path, getRoot returns null here.

  5. The toAbsolutePath() method returns the full path of the given relative path starting from the default root. In this case, the root is C:\ and so, it returns C:\temp\.\files . Observe that this path contains a . , which is redundant. This shows that the toAbsolutePath method does not return the canonical version of the path. To get that, we need to call normalize() . The normalize() method is practically the same as the getCanonicalPath method of the java.io.File class (except the fact that normalize() returns a Path object instead of a String ).

  6. The toRealPath method returns the canonical path for the given path and also checks if the file (or the directory) referred to by the path actually exists. If the file does not exist, it throws an IOException . Thus, in this example, if the files directory exists, it will return C:\temp\files and if it does not, it will throw an IOException . (In reality, a java.nio.file.NoSuchFileException is thrown but since the JavaDoc says it throws IOException , we will go with IOException . NoSuchFileException is an IOException .)
