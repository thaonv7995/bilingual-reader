To sum up all of the enhancements to the switch statement/expression in Java 21, a switch block may now have:

  • case constants - Examples: case 1, case "yes", case MONDAY
  • case null - Requires all switch statements and expressions to be exhaustive. null does not dominate anything. Nothing dominates null except default.
  • case patterns - Examples: case String s, case CheckingAccount ca
  • case patterns with guards - Examples: case String s when s.startsWith("-"), case Integer i when i==0
  • case patterns with record deconstruction - Example:

  • default - All kinds of switch statements and expressions can have the default branch. But in case of a switch with pattern matching, the default branch is allowed only if the switch is not already exhaustive. For example:

Pattern matching with records and sealed classes adds substantial complexity in applying the switch rules related to its exhaustiveness and dominance of case labels. One has to be a human Java compiler to determine if a given switch code is really exhaustive if it involves classes, interfaces, and nested records. This task however can be easily done by IDEs. Fortunately, the exam does not ask questions based on contrived code snippets. If you just understand the basic principles explained in this section and do a few mock questions, you will be well prepared for the exam.
