19.4 Writing to and reading from console☝

Conceptually, a console is the dashboard that allows a user to control various aspects of the system through various knobs and levers. In modern computers, a console is a text based window from where a user can interact with the Operating System by executing shell commands. On a Windows machine, it is the DOS prompt or the command prompt and on a Linux machine it is the command shell such as bash. Advancements in GUIs has made consoles nearly obsolete for end users but for programmers, consoles are indispensable during development and testing of programs primarily because they allow programmers to interact with the program without spending any time in developing a custom GUI. They provide an easy way for a program to output information to the programmer and also to accept input from the programmer while the program is executing.

A console is not available when you launch a program by double clicking an executable file (or by double clicking a file whose type is associated with an executable) from the file explorer. In such a case, the only way for the program to interact with the user is to create custom GUI windows using a windowing toolkit or an API. But when you execute a program from the command line, the program automatically gets access to that command window. It can write textual data to it and can collect user input through the same with very little effort. The same is true for a Java program as well.
