If you are instantiating a type safe ArrayList while assigning it to a type safe variable in the same statement, then you may omit the type name from the instantiation as shown below: Upon encountering <>, the compiler infers the type of the ArrayList object from the type of the variable it is being assigned to. The <> operator is known as the “diamond operator”, and was introduced in Java 7. It can be used only in a new statement. Thus, the statement ArrayList<> al; will not compile because it has nothing from which the compiler can infer the type of al. Although surprisingly, var al = new ArrayList<> ();, which too doesn’t have much information for the compiler to infer the type, compiles fine. The compiler infers the type of al as ArrayList<Object>. Similarly, the compiler can also infer the type argument in method invocations.
