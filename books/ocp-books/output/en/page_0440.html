Serialization and inheritance☝

If a class implements Serializable, all its subclasses automatically become serializable. Although there are ways to prevent serialization of such a subclass (not important for the exam), there is no way to make the subclass “unimplement” the Serializable interface.

The more important case is when a super class of a serializable class is not serializable. The rule is that if a serializable class extends a non-serializable class then the no-args constructor (and instance initializers) of the non-serializable super class will be executed.
