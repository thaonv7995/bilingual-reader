Files, Directories, and Folders☝

When you look at your file system through the file explorer, you see folders and files. Folders contain files and/or other folders, while files contain the data. Observe that a folder may contain multiple files and folders but a file or a folder can be contained exactly within one folder. This is what makes it a hierarchical file system. If you use the command line (or the shell), you can observe the same hierarchical structure in terms of files and directories using the dir command on Windows (or ls on Linux). At the top of a hierarchical file system is the root of that file system. In Windows, a root is denoted by a drive letter such as c: and in Linux, it is denoted by just a forward slash (/).

In the Java world, however, there is a single abstraction called "file" for files as well as directories. A file could be a regular file or it could be a directory (aka a folder). You can check what it is by inquiring its "attributes". So, if a file is a directory, you can invoke directory related operations on it such as retrieve the names of the files that it contains and if it is a regular file, you can read/write data from/to that file using various type of streams that you saw earlier.

File name☝

The name of a file is just a string that identifies the file within its parent directory. Conventionally, names for regular files end with a dot and a multi lettered extension such as .txt or .java while names of directories do not.

Pathname and Abstract pathname☝

An operating system uses a pathname to identify a file, while Java uses an abstract pathname for the same. The JVM converts an abstract pathname to the real pathname and vice-versa as and when required. The conversion logic depends on the operating system.

An abstract pathname is a series of zero or more file names separated by a system specific separator character. On Windows the separator character is back slash (\),
