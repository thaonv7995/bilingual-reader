Things get interesting if the finally block too has a return statement. Can you guess what will happen if I update the finally block in the above code to: finally{ System.out.println("In finally"); return 3; } . The getValue method is about to return a value 1 or 2 but the the finally block provides a return value of 3 . Which value should the method return to the caller? Java resolves this dilemma by mandating that the value returned by the finally block supersedes any value returned by the try or the catch blocks. The value returned by the try or the catch blocks are discarded.
