Note that, to maintain backward compatibility, a null value still does not match the default label. It matches only the null label. Without the case null block, a switch statement will still cause a NullPointerException to be thrown if the selector expression returns null.

5. Any reference type - With Java 21, the switch statement (and switch expression) has been enhanced significantly. This enhancement is commonly referred to as “pattern matching for switch”. Among other things, this enhancement allows the selector expression to return any reference type (including all wrapper types). Since there are several additional rules that apply in this case, I will discuss it detail in a separate section.

Compare this to an if/else statement where branching is done based on the value of a boolean expression. This limits an if/else statement to at most two branches. Of course, as we saw earlier, you can cascade multiple if/else statements to achieve multiple branches.

Case labels must consist of one or more compile time constants that are assignable to the type of the selector expression. For example, if your selector expression is of type byte, you cannot have a case label with a value that is larger than a byte.
