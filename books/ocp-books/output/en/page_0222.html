Let us make it a little more interesting. What if you want to log only FileNotException and ArrayIndexOutOfBoundsException and re-throw all other RuntimeExceptions? This is not that easy because if you put a catch clause that uses a super class of FileNotException and ArrayIndexOutOfBoundsException (which is Exception, by the way), it will catch all the four exception classes because Exception is a superclass of all them.

If you put a catch clause that uses a super class of NullPointerException and IllegalArgumentException (which is RuntimeException, by the way), it will catch ArrayIndexOutOfBoundsException as well. So, you will need at least three catch blocks.
