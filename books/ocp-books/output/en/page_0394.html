Blocking vs Non-blocking I/O☝

For a long time, Java had only the classes in java.io package to perform I/O related tasks. However, these classes implement what is known as “blocking” I/O operations. They are called blocking because once you call a method to read or write data using these classes, that call will block until the operation is complete or until the stream has ended. For example, if you try to read data from a remote file or from a network socket, the thread on which the read method is being executed may be blocked for a long time and will not be available to perform other tasks.
