Well, first thing that you should observe is the presence of { } between new Animal() and ;. This is not the syntax for instantiating a class. For instantiating Animal, you would have to write new Animal(); and if you try to do that you will get a compilation error that says, “Animal is abstract; cannot be instantiated”.

In fact, the above code uses the syntax for declaring as well as instantiating a concrete inner class that extends Animal without giving this class a name. Since no name is given to this class, it is called an anonymous class. When the compiler sees this code, it actually creates a class, gives this class a weird looking name, and generates a separate a class file for this class. It is this class that is instantiated at run time. Since Animal class doesn’t have any abstract method, this anonymous class doesn’t need to implement any method. Had Animal had an abstract method, you would have to implement this method inside the curly braces.

The same thing is happening in the case of the interface example above. The code is defining and instantiating, in a single statement, an anonymous class that implements the Dummy interface.

You are not expected to know all the nitty-gritties of anonymous classes for the exam. Just remember that anonymous classes are never abstract and there is no way to extend them. They cannot have an explicitly defined constructor either.
