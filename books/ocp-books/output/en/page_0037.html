The code looks quite straightforward. It tries to match the actual type of the object referred to by obj with the reference types given in the case labels. The block associated with the case label that matches with the type of the object is then executed. For each of the case labels, a pattern variable is also defined, which can be used within that case block. The switch statement with the old syntax works exactly the same way. Observe that it requires a break statement for each case block.
