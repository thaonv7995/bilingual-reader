Since the number of elements in the list is known, a Spliterator created over a List is “sized”. Therefore, it prints true and 5.

The trySplit method tries to split the existing elements in the Spliterator approximately in half. Since the exact splitting mechanism depends on the implementation, we cannot rely on which half will get how many and what elements.

The tryAdvance method returns true if the advance was successful. Therefore, the while loop and the call to forEachRemaining are basically doing the same thing.
