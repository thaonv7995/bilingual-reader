For example, here is a statement that you saw earlier for printing all the elements of a list: list.forEach( obj -> System.out.println(obj) ); It uses a lambda expression to create the accept method of the Consumer interface. In this lambda, all we are doing is creating a variable named obj and then passing it to the println method of System.out. The variable serves no purpose other than being a conduit to pass an object reference to the method. We don't even care about the name of the variable. 

Now, see this statement: list.forEach(System.out::println); Here, we are essentially telling the compiler to use the println method of System.out to generate the code for the Consumer object expected by the forEach method. The compiler knows that the generated Consumer's accept method will be passed an element of the list as an argument. Since println also expects one argument, the compiler can easily put two and two together and generate the code that passes the element received by the accept method down to the println method.

That's pretty much all that method references do. You tell the compiler which method you are interested in using the double colon syntax and if the method expects any arguments, the compiler will automatically pick up the values that are available in the context and pass them as arguments to the method that you are interested in. The only catch is that the compiler must be able to figure out exactly which method you want to call unambiguously and that it must be possible to call it with the information available in the context, which means the compiler must be able to identity the object on which the method has to be invoked (aka target reference) and the compiler must be able to identity the values that can be passed as arguments to the method.
