blocking I/O. If a thread is blocked on a read operation for even a few milliseconds, the server will grind to a halt very soon because there are only a limited number of threads. It is for this reason that older web servers used “thread pooling” to process requests.

To overcome this limitation, Java introduced a non-blocking I/O (aka NIO) library in Java 1.4 and updated it in Java 7 (aka NIO2). These classes are packaged in the java.nio package. Working with non-blocking I/O is substantially more complicated than working with blocking I/O but fortunately, this topic is not included in the exam. However, the nio library also includes the java.nio.file package which includes a few classes such as Files and Paths that are very helpful even while dealing with the blocking I/O and the functionality provided by this package is on the exam.
