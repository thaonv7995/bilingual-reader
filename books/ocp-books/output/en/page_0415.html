It is not necessary for an abstract pathname to refer to an actual file in the file system, but if you want to be able to convert it to a real pathname, every component except the last one of an abstract pathname must correspond to a directory. This makes sense because a file cannot contain other files, only a directory can.

Ideally, you should use the platform specific separator character while writing pathnames in your code (if you want to hardcode them in your code at all) but on Windows, the JVM automatically converts forward slashes to back slashes and so, if you use forward slashes in your code, the path will be acceptable on Windows as well as on Unix like machines. You will learn better ways to work with paths in a platform independent manner soon.

Absolute path and relative path☝

To access a file in a hierarchical file system, we need to know not just the name of the file but also the names of its containing folders, all the way up to the root. So, if a pathname includes all the names starting from the root down to the file that you want to access, it is an absolute path.

For example, c:\a\b\c\test.txt is an absolute path because we can start from the root c:\ and step inside the folder named a, then in b, and c to reach the test.txt file.

Similarly, on Linux, since the root is denoted by /, /a/b/c/test.txt would be an absolute path.

A relative path, on the other hand, is not sufficient in itself to identify a file in a file system. It can only identify a file when used in relation to another path whose location is already known.

For example, b\c\test.txt is not sufficient in itself to locate the test.txt file. However, if we are told that this path is relative to c:\a, then we can figure out its exact location. Thus, any path that does not start with a root is a relative path.

Current directory☝

The current directory is the path from where a program is run. For example, if you run your Java program from c:\temp, then c:\temp will be the current directory irrespective of where the class file being executed is located.
