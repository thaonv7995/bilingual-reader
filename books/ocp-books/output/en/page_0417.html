19.1.9 Using the java.io.File class☝

You have already seen most of the important classes of the java.io package. The only other class that you need to know about from this package is the File class. It is an immutable class whose objects represent a file or a directory path name. The purpose of this class is not to operate on the data inside the file but to work with the file itself in a platform independent manner.

Observe that a File object representing any file can be created irrespective of whether the file actually exists or not in the file system. The createNewFile method returns true if it was able to create an empty file in the file system. It could fail (and return false) if the file already exists or if there is a security restriction.  The renameTo , makeDir , makeDirs , and delete methods work similarly.
