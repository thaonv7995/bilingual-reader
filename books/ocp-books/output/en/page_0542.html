ConcurrentSkipListMap , ConcurrentSkipListSet , CopyOnWriteArrayList , and CopyOnWriteArraySet classes that provide similar thread safe implementations for Map , Set , and List . The exam does not expect you to know the details of all of these classes but it is good to be aware of them professionally as well as for interview purpose.

You can replace a HashMap with a ConcurrentHashMap in most cases but not when you need to store null key or values because ConcurrentHashMap does not allow those.

CopyOnWriteArrayList , on the other hand, does support null elements though.

Using synchronized wrappers for collections☝

The java.util.Collections class has static methods such as List synchronizedList(List l) and Map synchronizedMap(Map m) that wrap a given list or a map into a thread safe List or Map implementation class. Using these synchronized wrappers instead of the concurrent versions you saw above solves the three problems that I explained at the beginning but there are two issues with them.

  1. They provide thread safe check-and-update methods but they do not provide a thread safe way to iterate through the items. That’s our next topic.
  2. They synchronize all read and write operations using a single lock. This technique provides a lower performance as compared to the concurrent versions, which use better techniques internally to provide thread safety as well as performance.

Therefore, there is no compelling reason to use the synchronized wrappers instead of the concurrent classes except when you receive an thread unsafe collection (may be as a method parameter) and want to make it accessible through multiple threads.
