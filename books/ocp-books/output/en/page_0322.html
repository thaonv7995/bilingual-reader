Select 1 correct option.

A. walking or running?

B. Nothing will be printed.

C. Compilation error

D. Exception at run time

Correct answer is C.

This example illustrates the importance of the context in determining the target type of a lambda expression. The RunnerAndWalker class has two overloaded runOrWalk methods, which take objects of different interface types as arguments. The interfaces have exactly one abstract method that does not take any argument and does not return anything. Other than the names of the interfaces and the names of the methods, the two interfaces are functionally identical.

Now, when the compiler sees the lambda expression passed to the invocation of the runOrWalk method, it needs to identify the target type of that expression from the context so that it can invoke the right method. It needs to decide which interface is this lambda expression trying to implement. It checks the number of method parameters, their types, and then the return types of the methods to make a decision. Unfortunately, in this case, all of the three things match. The expression satisfies the needs of both the interfaces equally well.
