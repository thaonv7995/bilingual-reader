Not important for the exam, but it is good to know that a well formed paths do not end with a trailing separator character (unless it is referring to the root such as C:\ or / ). So, even if you try to create a Path with a trailing slash, it will be ignored. For example:

System.out.print(Path.of("/a/b/"));  will print /a/b  and not /a/b/ .

Using resolve and resolveSibling☝

Absolute paths on different environments in which an application runs, seldom match. For example, let’s say your application needs to read a properties file named values.properties .

On your Windows machine, you may be running this application under C:\dev\projects\fintech  and this file may be located at C:\dev\projects\fintech\props\values.properties . But in  production, the application may be running under /home/ec2-user/prod/fintech  and the actual location of this file may be /home/ec2-user/prod/fintech/props/values.properties . Obviously, if you hardcode the absolute path to this file in your code based on your development environment, the application will not be able to find this file in production. Therefore, it is advisable to use relative paths instead of absolute paths in Java code. The base path is provided to the application though a command line argument and the application “resolves” the relative paths (which are same on all environments) relative to the base path whenever it needs to locate the file in the file system.
