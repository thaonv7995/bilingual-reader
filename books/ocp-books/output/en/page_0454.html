The readLine method that takes no argument is unremarkable. It just returns the complete line of input entered by the user. I entered some input and the same is printed by the format method. The readLine method with a format string argument provides a textual hint to the user about what they need to enter. Observe that in both the methods, the keys pressed by the user are echoed to the console. That is why you see some input and bob written twice in the above screenshot.

The readPassword methods works similarly but they do not echo the keys pressed by the user to the console. This is useful when you want the user to enter sensitive information that should not be displayed on the console to prevent someone watching over the console from seeing it.

The data returned by the readPassword methods is not encrypted but note that these methods return a char[] instead of a String so that you can clear the char array (such as by using Arrays.fill method) after making use of the password. This ensures that the password typed by the user will not be revealed even by dumping program memory. It would not have been possible to do so if they returned a String because strings are immutable.
