This approach allows the application code to be written in a completely language neutral fashion and also allows the creation of the region specific resource bundles to be handled by a team that knows the local language. Such an application does not require any code change to be ported into different regions and is, therefore, called "internationalized". Refactoring an application to remove any region specific hardcoding from the application code is called "internationalization" or "i18n" in short (because there are 18 characters between i and n in the word internationalization) and the process of creating and packaging region specific files, i.e., resource bundles, for a particular region or language for the application is called "localization".

24.1.1 Detecting a user's Locale☝

A user's region (the word region and country are used interchangeably in this context) and language specific information is captured by the java.util.Locale class. It has several methods such as getCountry() and getLanguage() that return specific details of the region that it represents. You are not expected to know the actual values that these methods return for any locale, but be aware that most of these methods return well defined codes. For example, they return "en" and "US" for language and country if the Locale represents a region in the US that speaks English. Note that the language is a string of 2 to 8 letters in lowercase and the country is a string of 2 or 3 letters in uppercase.

A Locale also captures a "category" that tells you whether this locale information is meant for customizing UI displays or for customizing textual output generated by the application. This category is represented by a static nested enum defined in Locale named Category. It has just two constants - DISPLAY and FORMAT. Category is helpful when the application needs to make a distinction between the two types of customizations. For example, an application may want to display UI controls in local language Hindi but wants to generate reports in English.
