User's locale can be retrieved using the static methods getDefault() or getDefault(Locale.Category) of Locale. Many applications provide the user with an option to change their default locale. This can be done using Locale.setDefault(Locale newLocale) and Locale.setDefault(Locale.Category category, Locale newLocale). These methods change the default locale of the JVM and any subsequent calls to getDefault will return the new locale.

Observe that Builder's has a setRegion (and not setCountry) method but this API treats region and country as the same. Locale also defines several public static constants for countries and languages for example, Locale.US and Locale.CHINESE.

An important point to note about Locale is that it may not always have values for all of the details of the location that it represents. For example, Locale.CHINESE is a Locale that represents just the CHINESE language. Calling Locale.CHINESE.getCountry() will return an empty string (not null). Similarly, a calling getLanguage() on a Locale built using new Locale.Builder().setRegion("US").build() will return "".
