Ideally, I would like the above program to print either Id = 1 Name = Amy or Id = 2 Name = Bob. But imagine the following scenario:

The program starts the main thread. The main thread creates a new Student object and starts two new threads T1 and T2. The main thread then sleeps for 5 seconds (or joins on the two threads). Now, the OS schedules thread T1 to run. T1 executes the statement s.id = 1; but before T1 could execute the second statement, i.e., s.name = "Amy"; , the OS preempts T1 and schedules the thread T2 to run. The second thread executes two statements, i.e., s.id = 2; and s.name = "Bob" and terminates. The OS schedules T1 to run again. T1 executes the remaining statement, i.e., s.name = "Amy"; and terminates. So, now id is 2 and name is "Amy" . Finally, the main thread wakes up and since both id and name fields are volatile, the main thread is able to see the updated values of both the fields. Therefore, it prints Id = 2 Name = Amy.
