Here are a few points worth noting from the above code about the new syntax:

1. uses arrow -> instead of colon :

2. single expression statement or block: Every switch label is associated with either a single expression statement or single a code block (i.e. a set of statements enclosed within { and }). If you use a single statement, you cannot write a break statement after that because, well, then you are not writing a single statement!

3. No break statements: The new syntax ensures that the switch statement breaks automatically after the expression or the block executes. It is valid to write a break statement in the code block but it is redundant because the control will exit out of the switch statement after completing the code block anyway. In other words, the new syntax does not have fall through behavior. In fact, it is not possible to simulate the fall through behavior using the new syntax.

4. Multiple case values in the same case label: The new syntax has only one way to allow multiple values to be associated with a case block as shown in this line above: case 2, 3 -> System.out.println("Two or three arguments");.

5. Cannot mix syntaxes: Cannot mix old and new syntax in the same switch statement. You can, of course, mix them in the same code base.
