Getting file attributes☝

The File class lets you enquire about its properties of the file through methods such as isDirectory() , isFile() , isHidden() , and lastModified() .

The File class has been in existence since Java 1.0 and is usually sufficient for routine things.

However, since the exam does not focus on the File class much, I am not going into the details of all its methods. I suggest you to browse through the JavaDoc for the File class and write a few test programs to see how they work.

19.2 The java.nio.file package☝

The java.nio.file package provides a more comprehensive and platform independent access to the file system than the java.io.File class. It does so, by formally abstracting the concepts of file system, path, and file attributes into a large set of interfaces and classes. Some of the things that you can do using this package that you cannot do using the File class are - open files in ten different read/write modes, copy files and directories, refer to files in a different file systems, watch directories for changes at run time, get finer control over symbolic links, recursively traversing directories, synchronize writes to the file system, and managing file attributes. Yes, that’s a lot of things and there are more! But do not worry. You are not expected to know all of these things. Usually developers learn how to perform these tasks only when they actually need them on a project.

The OCP exam also does not expect you to know about most of the above mentioned features and focuses only on one feature - working with paths and their properties.
