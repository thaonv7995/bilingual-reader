15.2 Understanding checked exceptions, unchecked exceptions, and errors

15.2.1 Checked and Unchecked exceptions☝

As discussed earlier, exceptions thrown by a method are part of the contract between the method and the user of that method. If the user is not aware of the exceptions that a method might throw, they will be blindsided during run time because their code would not be prepared to handle that exception. The throws clause of a method is meant to list all exceptions that the method might throw. If an exception is listed in the throws clause, the user of the method will know that they need to somehow handle that situation.
