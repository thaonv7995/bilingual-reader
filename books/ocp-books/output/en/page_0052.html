1. The type of the option variable is Object. The regular switch (i.e. switch without pattern matching) does not accept Object as a selector expression type. It accepts only a limited set of types, which includes String. So, you have to either convert the case labels into case labels with pattern matching (with or without guards, such as case String s: or change the type to String. Let us change it to String.

2. You cannot mix the old and the new syntax in the same switch. Therefore, let's change -> to :. (As an exercise, change : to -> and then find out the issues.)

3. The usage of case null implies that you are using the new features of switch and that requires a switch to be exhaustive. So, either remove this case or add the default label. Either option will do.
