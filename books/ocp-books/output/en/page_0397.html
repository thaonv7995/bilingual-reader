For example, the file that you are trying to open may not be present or the file that you are trying to write to may be marked read only by the OS. There are several specialized exception classes such as FileNotFoundException or EOFException defined in the standard library for this purpose, but all of them are rooted under the checked exception class java.io.IOException and so, the throws IOException clause in main satisfies all of them.

  5. Finally, we use the close() method to close the input and output streams. Closing I/O streams using the close() method is important because it allows any system resources associated with the streams such as file locks to be reclaimed by the OS. Closing an output stream also flushes all data remaining in the output pipeline to the destination. Since InputStream and OutputStream implement AutoCloseable, the preferable approach is to let the try-with-resources statement close the streams automatically.

The Java API designers decided to return an int from read() instead of byte because an extra value was needed to signify the end of the stream. The total number of values returned by the method is 257, which includes 256 values (00000000 to 11111111) represented by a byte and an extra value of -1 to denote the end of the stream. Since a byte cannot represent 257 values, a larger data type was needed. This could have been achieved using a short as well, but int was used to remain consistent with the C language.
