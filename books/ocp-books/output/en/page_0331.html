The JDK standard libraries are littered with interfaces with exactly one abstract method, and technically, they are all functional interfaces but but not all of them are annotated with @FunctionalInterface. Some of them, such as java.lang.Comparable, are not marked with @FunctionalInterface because they define intrinsic behaviour of classes and do not define pure independent functionality. So, conceptually, they are not “Functional Interfaces”.

17.1.6 Using Functional Interfaces with Collections API☝

Generics made Java collections type safe, while Lambda expressions together with functional interfaces gave them super powers. Most of the classes and interfaces of the collections API were updated in Java 8 to include methods that accept functional interfaces so that programmers can invoke those methods with lambda expressions. Things such as iterating, filtering, and replacing elements, that took several lines of code can now be done by half a line of code.
