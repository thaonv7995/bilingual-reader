The JVM knows what kind of object f1 actually points to and it will not let the cast of f1 to Mango succeed. The JVM will throw a ClassCastException when it executes this line.

The JVM is essentially the second line of defence against violation of Java's type safety. It checks at run time what the compiler is unable to check at compile time. If the JVM sees that a reference variable is being cast to a type that does not satisfy the is-a test for the class of the object to which the variable is referring, it will throw a ClassCastException. Thus, the type safety of the program is never compromised.

Casting a reference to an interface☝

It is easy for a compiler to determine whether a reference of one class can ever point to an object of another class because a class can only extend one class at the most. Therefore, if the class mentioned in the cast is a subclass of the declared class of the reference, the compiler knows that the cast may succeed at run time.
