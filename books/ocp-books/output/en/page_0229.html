Most significantly, a lot of file I/O classes already implemented the java.io.Closeable interface even before Java 7. Java 7 made java.io.Closeable extend java.lang.AutoCloseable , thereby making all those classes amenable to automatic resource management in one fell swoop. Observe that AutoCloseable is in java.lang package because it is not specific to I/O resources. In other words, as far as Java is concerned, any class that implements AutoCloseable is a resource that can be managed automatically.
