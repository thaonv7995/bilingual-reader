In other words, Java doesn't allow its serialization mechanism to be used to serialize any random object unless the developer explicitly accepts the risks involved by implementing the Serializable interface. This is not to say that a serializable class is always a security risk. You may write additional custom code on top of Java's serialization mechanism to mitigate the risks associated with serialized data. These are described in detail in "Secure Coding Guidelines for Java SE" document published by Oracle. Although it is not required for the OCP exam, I strongly encourage you to go through it.

The ObjectOutputStream and ObjectInputStream classes☝

The java.io.ObjectOutputStream and java.io.ObjectInputStream classes contain the logic of serializing and deserializing objects respectively. ObjectOutputStream's writeObject method takes a serializable object and converts it into a stream of bytes. This stream of bytes can be connected to any other output stream such as a FileOutputStream, as done in the above program. Similarly, ObjectInputStream's readObject method reads a stream of bytes and creates the object back. This stream of bytes can be sourced from any other input stream such as a FileInputStream, as done in the above code. Just like other stream methods, these methods too throw IOException.
