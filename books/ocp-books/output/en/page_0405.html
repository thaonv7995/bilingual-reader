Observe the following points in the above code and output:

  1. A total of three lines are read from the file. The third line is an empty string (it is not null). This is because a line is considered to be terminated by any one of a line feed (\n ), a carriage return (\r ), a carriage return followed immediately by a line feed (\r\n ), or by reaching the end of the file. Note that \n is not a string containing two characters. It is just one character whose numeric value is 10. Similarly, \r is just one character whose numeric value is 13. When you write these characters in a file, they show up as line breaks in a text editor. To prevent the compiler from treating them as line breaks in our source code, we use escape sequences \r and \n . In the above code, the first line is terminated by \r\n , the second line is terminated by \n , and the third line is terminated by \r . There is no fourth line in the file.
  2. We keep on reading the lines until the BufferedReader returns null .
  3. We are using the try-with resources statement with multiple resources and var  declarations. This is typical while using I/O streams.
