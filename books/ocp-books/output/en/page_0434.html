Java Serialization is readymade mechanism available in the standard Java library for capturing the state of an object in a form that can be saved and can be used later to restore an object to that same state. This mechanism was in vogue a decade ago before the advent of third party libraries that could do the same thing using cross-platform formats such as JSON and XML. Practically, Java serialization has completely fallen out of favor and is almost never used nowadays because not only is it tightly coupled to Java, it is also a security risk. Unfortunately, the OCP certification exam still expects you to know about this mechanism in detail, most likely because countless libraries and frameworks depend directly or indirectly on this mechanism. As usual, I will start with a simple example to give you an overview and then introduce the complications.
