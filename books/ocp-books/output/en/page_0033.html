Now, coming to the subtle aspects of switch expression, which are not evident in the above code.

Value of a switch expression☝

Remember that an expression must have a value. Since a switch expression is an expression, it must also always have a value. This implies that the code inside a switch expression must generate a value for every possible value of the selector expression. Therefore, you cannot have a switch expression that has an execution path that doesn't generate any value. For example, the following expression is invalid.

return income * switch(taxBracket){
  case 0 -> 0.1;
  case 1 -> 0.2;
};

What will happen if taxBracket is neither 0 nor 1 ? Well, there is no instruction about what to do in that case and thus, the given expression cannot be evaluated if the value of taxBracket is neither 1 nor 2 ! To avoid this issue, Java mandates that a switch expression must be "exhaustive", which means, it must provide an execution path that returns a value for every possible value of the selector expression.
