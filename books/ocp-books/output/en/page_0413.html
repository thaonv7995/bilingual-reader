There are a total of 12 characters in test1.txt (don’t forget the two characters used for terminating the first line). In the first iteration of the while loop, the read method fills buff with the first 5 chars, i.e., hello. Observe that we have not specified the offset and the length arguments to the write method. So, it writes all of the characters in buff to test2.txt.

Thus, hello is written to the file. In the second iteration, the read method reads the next 5 chars, i.e., \r\nwor, which are then written to the file. Now there are only 2 more characters left to be read from the file. Thus, in the third iteration, the read method reads only those characters and puts them in the first two locations of buff. The remaining three locations in buff are unaffected and still contain the characters read in the previous iteration, i.e., wor.

So, buff now contains ldwor. Thus, the write method writes ldwor to the file. The fourth iteration does not happen because the read method returns -1.

Had we used fw.write(buff, 0, count) instead of fw.write(buff), the program would have copied the contents correctly.
