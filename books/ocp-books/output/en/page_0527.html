Here are a few things that you must be aware of while using a ReadWriteLock :

  1. It has only two methods - Lock readLock() and Lock writeLock().
  2. Since ReadWriteLock is just an interface, several features of the lock such as acquisition order, reentrancy, and fairness, depend on the implementation class, i.e., the ReentrantReadWriteLock class.

Here are a few things that you must be aware of while using a ReentrantReadWriteLock :

  1. Locks acquired using a ReentrantReadWriteLock are reentrant.
  2. Lock upgrading is not allowed - a thread that already has the read lock will not be able to acquire the write lock unless it releases the read lock first. Calling tryLock() on the write lock would just return false and calling lock() on the write lock will wait forever if the thread does not release the read lock first.
  3. Lock downgrading is possible - a thread that has the write lock may acquire a read lock without releasing the write lock. However doing so, will not automatically release the write lock. It would have to release both the locks separately.
  4. The ReentrantReadWriteLock class defines several useful methods such as getReaderHoldCount, getReaderLockCount, isWriteLocked, and isWriteLockedByCurrentThread.
