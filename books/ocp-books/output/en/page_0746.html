Since the default locale is en_US, whenever you request a bundle without specifying the locale (as in //1) or for a locale for which no locale specific properties file is present (as in //3, //4, //5, and //6), the bundle for the default locale, which is created using msgs_en_US.properties will be returned. Thus, //1, //3, and //4 will print Hi, and //5 and //6 will print Regards.

2. Asking for a bundle for es_MX as in //2 and //7 will return the es_MX bundle because msgs_es_MX.properties is present. The default bundle will not be used at all if you look for keys in this bundle. Therefore, //2 will print Hola but //7 will throw a MissingResourceException because the es_MX bundle doesn't contain the key regards.

In the exam, you may encounter a badly written question that does not explicitly specify the default locale of the application. In such a case, you may assume the default locale to be en_US but only if it is absolutely necessary to arrive at the correct answer.
