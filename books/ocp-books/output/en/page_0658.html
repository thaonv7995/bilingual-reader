The LocalDateTime object can be translated to a LocalDate by dropping the time component, which means both the LocalDate instances contain the same date and so, the first print statement prints 0. However, since a LocalDate doesn't have the time component, it can't be translated to a LocalDateTime. Therefore, the second print statement throws a DateTimeException. Ok, now how about the following?

Before answering, remember that LocalDateTime doesn't have zone information and so, it can't be converted into a ZonedDateTime. Therefore, the second print statement will throw an exception.

In the previous section you saw how a ZonedDateTime and an OffsetDateTime can be translated to an Instant but the reverse is not possible. Therefore, for example, ChronoUnit.HOURS.between(Instant.now(), ZonedDateTime.now())); will work and return 0 but ChronoUnit.HOURS.between(ZonedDateTime.now(), Instant.now()); will throw a DateTimeException at run time.
