Tips to work out code containing switch

The new enhancements to switch have truly turned it into a monster. It is extremely powerful but its syntax has so many ifs and buts that it is not possible for normal humans to remember and apply all of the rules and figure out how a switch will behave during an exam!

My goal here is not to reproduce all the rules of switch, which you can find by going through JEP 441 if you like, but to show you only those rules that are essential for correct development and for answering questions in the exam.

Based on experience, I can tell you that exam does not try to trick you with weird switch code but expects you to know and apply the following points to arrive at the answer:

1. Check whether the switch uses pattern matching or not by looking at two things. First, if the type of the switch selector expression is anything other than the legacy types, i.e., String, byte/Byte, char/Character, short/Short, int/Integer, or enums, it is a switch with pattern matching. Second, if the case labels use pattern matching expressions such as case String s, it is a switch with pattern matching. A switch with pattern matching must be exhausive, must follow the rule on dominance of case labels, and must not exhibit fall through.
