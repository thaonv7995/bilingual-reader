The above program is a good starting point because even though it is fully functional, it has several shortcomings that necessitate the need for many of the important classes and interfaces of Java I/O API. Observe the following points about the above program:

  1. The concept of raw input stream and raw output stream is represented by abstract classes InputStream and OutputStream respectively.
  2. Since the source and destination of our input and output streams are files, we are using FileInputStream and FileOutputStream classes provided by the Java I/O library to read from and write to the files. They extend InputStream and OutputStream respectively.
  3. The source of our input is a file located at c:\temp\test1.png and the destination of our output is another file located at c:\temp\test2.png . Note that these paths are Windows specific. If you are on Linux, you will have to use something like /home/myname/temp/test1.png . If you are thinking that the paths are totally making the program platform dependent, then you are right. They indeed are. There is a java.io.File class that helps us avoid platform specific path separators in the code. Furthermore, there is a Path API in java.nio.file package that helps us manipulate paths and interact with the file system in a platform independent manner.
  4. Although the read() method reads a byte from the input stream, the return type of this method is int , which is why the variable bite has been declared to be of type int .
