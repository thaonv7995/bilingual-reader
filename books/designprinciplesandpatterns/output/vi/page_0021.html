Hình 2-23 cho thấy cách phá vỡ chu trình bằng cách thêm package mới. Các class mà CommError cần được rút ra khỏi GUI và đặt vào package mới tên MessageManager. Cả GUI và CommError đều được buộc phụ thuộc vào package mới này.

Đây là ví dụ về cách cấu trúc package có xu hướng rung lắc và dịch chuyển trong quá trình phát triển. Các package mới xuất hiện, và các class chuyển từ package cũ sang package mới, để giúp phá vỡ chu trình.

Hình 2-24 cho thấy hình ảnh trước và sau của kỹ thuật kia để phá vỡ chu trình. Ở đây ta thấy hai package bị ràng buộc bởi một chu trình. Class A phụ thuộc vào class X, và class Y phụ thuộc vào class B. Ta phá vỡ chu trình bằng cách đảo ngược phụ thuộc giữa Y và B. Việc này được thực hiện bằng cách thêm interface mới, BY, vào B. Interface này có tất cả các method mà Y cần. Y sử dụng interface này và B triển khai nó.
